# cockpit-shell Specification

## Purpose

定义 DataWeave 的全站三栏框架（左侧功能分组菜单 + 中部观测台 + 右侧常驻 Agent）、功能分组菜单与驾驶舱健康中心首页的行为约束。

## Requirements

### Requirement: 三栏应用骨架

应用 SHALL 采用「左侧功能分组菜单 + 中部观测台 + 右侧常驻 Agent」三栏布局，作为全站统一框架。中部观测台 SHALL 承载各模块路由内容，左侧与右侧在所有页面持续可见（右侧可收起，见 `copilot-rail`）。

#### Scenario: 任意页面均呈现三栏框架

- **WHEN** 用户打开任意模块路由（如 `/`、`/tasks`、`/fleet`）
- **THEN** 页面同时渲染左侧分组菜单、中部该路由内容、右侧 Agent 面板（或其收起态悬浮球）

#### Scenario: 中部随路由切换而菜单与右舷保持

- **WHEN** 用户点击左侧菜单切换模块
- **THEN** 仅中部观测台内容更新，左侧菜单与右舷 Agent 的状态不被重置

### Requirement: 功能分组菜单

左侧菜单 SHALL 按功能域分组展示：「数据研发」（数据集成、数据开发、调度运维）、「数据资产」（资产目录、数据血缘、数据质量、指标体系）、「资源与诊断」（集群机器、资源监控、智能诊断）。MVP 闭环路由（数据开发、调度运维、集群机器、智能诊断）SHALL 可导航；其余项 SHALL 作为占位展示且标注「即将上线」，不导致路由错误。

#### Scenario: 分组与图标渲染

- **WHEN** 菜单渲染
- **THEN** 各功能组以分组标题分隔，每项带 hugeicons 图标与中文标题，当前路由项高亮

#### Scenario: 占位项不可误导航

- **WHEN** 用户点击占位菜单项（如「数据集成」）
- **THEN** 进入占位页并提示「即将上线」，不抛错、不出现空白崩溃

### Requirement: 驾驶舱健康中心首页

根路由 `/` SHALL 呈现驾驶舱（健康中心）：展示「今日任务运行概况（成功/失败/运行中数量）」「失败任务列表」「Agent 正在诊断中的事项」三个区块，把自诊断能力前置到首页。

#### Scenario: 开屏即见全局态势

- **WHEN** 用户访问 `/`
- **THEN** 页面展示今日运行概况计数、失败任务清单、诊断中事项三区块

#### Scenario: 从失败项直达诊断

- **WHEN** 用户在驾驶舱点击某条失败任务
- **THEN** 进入该任务的诊断视图（见 `self-diagnosis`），且右舷 Agent 上下文同步到该失败实例
