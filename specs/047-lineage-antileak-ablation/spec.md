# Feature Specification: 抗泄漏消融——同规模下改训练能否修记忆泄漏（041-R 方案 B）

**Feature Branch**: `047-lineage-antileak-ablation`

**Created**: 2026-07-05

**Status**: Draft

**Input**: User description: "方案 B 抗泄漏消融：B1 真实表名增强 / B2 弃权训练。数小时 GPU 的真实验、产出不确定，但能把论文从『失败』升级成『补救也失败』。"

## 背景与动机 *(为什么做这件事)*

041-R 已实证的负结果：合成-only 训练的 1.5B 小模型在合成 held-out 上近乎完美（precision 0.995、幻觉 0.0009），却在真实 OOD ETL 脚本上急剧退化（真实 precision 0.270、幻觉 0.153、方向 0.496）；其失败机制以**记忆泄漏**为主——在解析不了的真实脚本上**逐字背诵合成训练分布的表名**（逐字泄漏 22.4%，JVM 上 40.8%）。逐规模曲线（0.5B→3B）证明泄漏随容量单调缓解，但 0–3B 内不足以追平大模型，且**读写方向混淆是与规模无关的另一独立病**。

一个诚实的负结果论文必须回答审稿人必问的一句话：**"你只是没用对训练方法吧？换个常规补救不就好了？"** 方案 B 就是**在同一 1.5B 规模、同一评测口径下**，实测两条最直觉的抗泄漏补救到底能不能修这个病。无论结果是"修好了"还是"修不动"，都是**决定性的、可发表的**测量：

- 若**修不动** → 论文从"合成-only 会失败"升格为"**连显而易见的补救也失败**"，负结果更硬、贡献更大。
- 若**部分修好** → 诚实报告"泄漏可被 X 补救缓解 N 个百分点，但方向病/domain-shift 仍在"，给出务实的边界，同样有价值。

**本特性的成功 = 产出决定性且诚实的测量，而非补救必须奏效。** 这一点贯穿全部成功标准。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - B1 真实表名增广消融 (Priority: P1)

作为论文作者，我要在合成训练数据里**注入真实世界的表名**（打散合成生成器的命名分布，让模型无法靠"背诵少数合成 schema/base 词"蒙混），在**其它配方完全不变**（同 SEED / LoRA r16α32 / 2 epoch / max2048 / 同 base 1.5B）的前提下重训一版模型，然后用**与基线完全相同的真实金标与评测器**测它，量化"真实表名增广能否降低真实脚本上的逐字记忆泄漏"。

**Why this priority**: 记忆泄漏是全篇立论基石，B1 直接攻击其最可能的根因（训练命名分布过窄）。这是审稿人最先想到的补救，必须先答。单独做完 B1 就已经能给论文补上"我们试了增广，结果是 ___"这一决定性段落，构成独立 MVP。

**Independent Test**: 仅训练 B1 一版权重、跑 `eval_real.py` + `leak_analysis.py` 于同一 `real.jsonl`（n=139/非空 59），得到一行可直接并入 041-R 泄漏对照表的数字（逐字泄漏率、合成形态率、真实 precision/方向），即完成，不依赖 B2。

**Acceptance Scenarios**:

1. **Given** 基线训练配方与合成数据管线，**When** 只把训练集的表名分布替换/混入真实表名池（配比与替换策略在数据模型中固化、可复现），其余超参逐字不变，**Then** 产出一版 B1 权重，且训练配方与基线的差异**仅在训练数据表名分布**这一个维度（消融纯净性）。
2. **Given** B1 权重，**When** 用与基线同一 `real.jsonl` 金标、同一 `eval_real.py`/`leak_analysis.py` 评测，**Then** 得到 B1 在真实集上的逐字泄漏率、合成形态率、precision/hallucination/方向，与基线 1.5B 行**逐列可比**。
3. **Given** B1 与基线结果，**When** 汇报，**Then** 明确给出"逐字泄漏从 22.4% 变为 X%"的方向与幅度，并**同时**报告合成 held-out 是否退化（增广不应偷偷牺牲合成能力去换真实数字，须如实披露 trade-off）。

---

### User Story 2 - B2 开放域弃权训练消融 (Priority: P2)

作为论文作者，我要给训练集**掺入"开放域负样本"**（非 ETL 或无字面表名的脚本，其标签为空 `{"reads":[],"writes":[]}`），教模型在**看不懂 / 无把握时输出空（弃权）而非硬抽/背诵**，在其它配方不变前提下重训一版模型，量化"弃权训练能否在真实脚本上降低幻觉/泄漏、提高 precision"。

**Why this priority**: 弃权是第二直觉的补救——负结果显示模型的病是"在 OOD 上乱背"，弃权正对症。相对 B1 略次，因它可能以牺牲 recall 换 precision，trade-off 更需谨慎解释；但它独立可测，且与 B1 正交（一个改命名分布、一个改弃权行为），单独完成即为论文补上第二个补救段落。

**Independent Test**: 仅训练 B2 一版权重、跑同一评测，得到 precision/hallucination/recall/方向 + 泄漏率一行，可与基线、B1 并列，不依赖 B1 完成。

**Acceptance Scenarios**:

1. **Given** 基线配方，**When** 按固定配比掺入带空标签的开放域负样本、其余超参不变，**Then** 产出一版 B2 权重，消融维度**仅在"是否含弃权负样本"**。
2. **Given** B2 权重，**When** 用同一真实金标评测，**Then** 报告 precision / hallucination / **recall（弃权最可能牺牲的指标）** / 方向 / 逐字泄漏，与基线逐列可比。
3. **Given** B2 结果，**When** 汇报，**Then** 显式给出 precision↑ 与 recall↓ 的**权衡量级**，不得只报有利指标；若弃权把模型退化成"几乎总弃权"（recall 崩塌换来的虚高 precision），须如实指出该补救不可用。

---

### User Story 3 - 消融综合裁决并入论文 (Priority: P3)

作为论文作者，我要把 B1/B2（及已完成的"更大模型 = 3B 点"一支）的结果综合成一张**抗泄漏消融表**并入 041-R 证据链与 limitations，给出统一结论：同规模下改训练**能/不能**修记忆泄漏与 domain-shift，以及方向病是否依旧无解。

**Why this priority**: 综合裁决依赖 B1、B2 至少之一完成，故排 P3。它把散点实验收敛成论文可用的一段论述与一张表，是"升级论文"这一初衷的最终落地。

**Independent Test**: 存在一张把 基线 / B1 / B2 / 3B 放在同一真实金标同一指标口径下的对照表 + 一段诚实结论（含 trade-off 与仍未解的病），并入 `paper-negative-result-findings.md`，即完成。

**Acceptance Scenarios**:

1. **Given** B1、B2 至少之一的评测产物，**When** 汇总，**Then** 生成一张消融表（行=基线/B1/B2/3B，列=真实 precision/幻觉/recall/方向/逐字泄漏/合成形态 + 合成 held-out 参照），所有数字来自同一评测器同一金标。
2. **Given** 消融表，**When** 写结论，**Then** 明确回答"补救是否奏效"，并保留对**方向病与 domain-shift 仍在**的诚实披露；结论与数据一致，不夸大任一补救。

---

### Edge Cases

- **补救"奏效"到翻转论文主结论**（例如 B1 让小模型真实 precision 追平 m2）：不得隐瞒或淡化——如实上调结论、重写 thesis 为"合成-only 会失败**但真实表名增广可救**"，负结果的诚实性高于"结论好看"。
- **补救使合成 held-out 崩塌**：增广/弃权可能牺牲合成能力换真实数字；必须**同时**报告合成 held-out 指标，避免"只看真实数字变好"的误读。
- **B2 退化为恒弃权**：若模型学会"永远输出空"骗高 precision，recall 崩到接近 0——须识别并判定该补救**不可用**，不得只报 precision。
- **样本量导致的统计噪声**：真实非空集仅 59（JVM 28），补救带来的小幅变化可能落在噪声内；须对比幅度与已知规模曲线台阶（如 22.4%→10.9% 是 1.5B→3B 跨一档），避免把噪声当效果。
- **消融不纯净**：若为做 B1/B2 顺手改了别的超参，消融失去可比性；每一支必须与基线**只差一个维度**。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供 B1 训练数据变体——在合成训练集中注入真实世界表名池以打散命名分布；真实表名来源、注入配比、替换/混入策略 MUST 固化为确定性、可复现（固定 SEED，无网络依赖或依赖已落盘的采集产物）。
- **FR-002**: 系统 MUST 提供 B2 训练数据变体——掺入带空标签 `{"reads":[],"writes":[]}` 的开放域负样本；负样本来源与配比 MUST 固化、可复现。
- **FR-003**: B1、B2 每一支的训练配方 MUST 与基线**仅相差一个维度**（B1=训练数据表名分布；B2=是否含弃权负样本），其余（SEED / LoRA 超参 / epoch / max-len / base 模型 / SYSTEM_PROMPT）逐字一致，以保证消融纯净、结果可归因。
- **FR-004**: 系统 MUST 复用既有评测器（`eval_real.py`、`leak_analysis.py`，必要时 `jvm_slice_eval.py`）与既有真实金标（`real.jsonl` n=139/非空 59，可选 `real-jvm.jsonl`），不得为 B 另造评测口径——B 的数字必须与 041-R 基线表逐列可比。
- **FR-005**: 每一支 MUST 同时产出**真实集**指标（precision/hallucination/recall/direction）**与合成 held-out** 指标，以暴露"以合成能力换真实数字"的 trade-off。
- **FR-006**: 每一支的记忆泄漏度量 MUST 沿用既有判据（幻觉表名中逐字命中确定性合成生成池的比例 + 合成形态率），使"泄漏是否被修"的判断与基线同源、gold-无关。
- **FR-007**: 系统 MUST 产出一张抗泄漏消融对照表（基线 / B1 / B2 / 3B 同口径）并入 `specs/041-script-lineage-extraction/paper-negative-result-findings.md` 的证据链，附诚实结论与 trade-off/未解病披露。
- **FR-008**: 新增权重 MUST 落在 worktree 外的 `weft-lineage-weights/`（经 `WEFT_WEIGHTS_DIR` 或既有 `parents[K]` 解析），不得进入 worktree 或被 git 跟踪；采集到的第三方真实脚本/表名池遵循 041-R 既有约定，默认不公开发布。
- **FR-009**: 汇报 MUST 对每一支给出补救的**方向与幅度**，并在补救使任一指标恶化（合成退化、recall 崩塌）时**主动披露**；严禁只报有利指标。
- **FR-010**: 若任一支的结果翻转论文主结论，系统 MUST 如实上修 thesis 与结论，诚实性优先于结论稳定性。

### Key Entities *(include if feature involves data)*

- **B1 训练集变体**：基线合成训练集 + 真实表名注入；关键属性=真实表名来源、注入配比、替换策略、SEED；与基线唯一差异维度=表名分布。
- **B2 训练集变体**：基线合成训练集 + 开放域空标签负样本；关键属性=负样本来源、配比、SEED；与基线唯一差异维度=弃权负样本占比。
- **B1/B2 权重**：各一版 merged 1.5B 权重，落 `weft-lineage-weights/`（gitignore），供评测/可选发布。
- **抗泄漏消融表**：行=基线/B1/B2/3B，列=真实 precision·幻觉·recall·方向·逐字泄漏·合成形态 + 合成 held-out 参照；单一评测器、单一金标产出。
- **既有真实金标**：`real.jsonl`（139/非空 59）、可选 `real-jvm.jsonl`（141/非空 28）——B 复用、不改。

## Success Criteria *(mandatory)*

> 成功 = **决定性且诚实的测量**，非补救必须奏效。以下标准全部以"测出并如实汇报"为准，不以"数字变好"为准。

### Measurable Outcomes

- **SC-001**: 至少完成 B1 与 B2 之一，产出一版消融权重及其在同一 `real.jsonl` 上的完整指标（precision/hallucination/recall/direction）+ 逐字泄漏率 + 合成形态率，全部与基线 1.5B 行逐列可比。
- **SC-002**: 每一支均同时报告真实集指标与合成 held-out 指标，trade-off（若有）被显式量化——不存在"只报真实、隐去合成退化"的单边汇报。
- **SC-003**: 每一支与基线的训练配方差异被验证为**恰好一个维度**（消融纯净性可核验：其余超参逐字一致）。
- **SC-004**: 存在一张 基线/B1/B2/3B 同口径抗泄漏消融表并入 `paper-negative-result-findings.md`，附一段与数据一致的结论，明确回答"同规模改训练能否修记忆泄漏"，并保留对方向病与 domain-shift 的诚实披露。
- **SC-005**: 全过程可复现——B1/B2 数据变体由固定 SEED + 落盘产物生成，任何人重跑得到同一训练集；评测复用既有管线无新口径。
- **SC-006**: 任一支若恶化或翻转结论，均被如实汇报（含 B2 恒弃权、B1 合成崩塌、补救翻转 thesis 等边界情形的显式判定）。

## Assumptions

- **同规模 1.5B、显存轻松**：B1/B2 均在 1.5B bf16 + LoRA 上跑（与基线同规模），12G 单卡数小时可完成，无需 QLoRA/量化（避免量化不一致污染对照）。
- **复用既有资产**：合成数据管线（`data/synth_pipeline.py`、`data/templates.py`、`data/augment.py`）、评测器（`realeval/*`）、真实金标（`realeval/gold/*.jsonl`）、基线权重与曲线（`weft-lineage-weights/run*`）均已就绪，B 在其上做最小增量。
- **真实表名池来源**：B1 的真实表名优先取自已采集的真实脚本金标/候选池（`realeval` 采集产物）或公开数据仓建模命名约定，遵循 041-R 第三方数据不默认公开发布的约定。
- **基线数字冻结**：对照基线取 `paper-negative-result-findings.md` 已实测冻结值（1.5B 真实 precision 0.270 / 逐字泄漏 22.4% / 方向 0.496；3B 逐字泄漏 10.9% 等），B 只新增行、不重跑基线。
- **评测口径不变**：沿用金标约定 A（仅字面出现于可执行语句的表；动态名/路径/临时视图/注释/配置驱动范围外），保证 B 与既有全部实验可比。
- **产出不确定性被接受**：本实验产出方向未知（可能修不动、可能部分修好、极小概率翻转），三种结果都视为成功交付，前提是测量决定性且披露诚实。
- **JVM 为可选扩展**：B 主体在 py/sh（`real.jsonl`）上做；是否把 B1/B2 也在 `real-jvm.jsonl` 上复评为可选加强，不作为完成门槛。
