# Phase 0 Research: 抗泄漏消融（041-R 方案 B）

解决 spec 遗留给 plan 的设计决策。每条给 Decision / Rationale / Alternatives。全部以**消融纯净**（每支与基线仅差一维）与**诚实测量**（不刷指标）为最高约束。

---

## 决策 1：B1 真实表名增广的注入策略

**Decision**：**替换而非稀释**——把训练集表名池从"合成生成名 400（`WAREHOUSE_SCHEMAS×BASES×SUFFIXES`）∪ HF 收割名"改为**真实表名主导**：把 HF 收割量大幅上调（`harvest_hf_names(limit_rows)` 4000→~20000，去重后目标真实名 ≥ 2000），并把 `synth_table_names` 的贡献从 400 降到**约 40（10%）**。训练所用表名 ≈ 90% 真实世界名 + 10% 合成名。其余管线（模板、渲染、split 隔离、列名池）逐字不变。

**Rationale**：记忆泄漏的根因假设是"合成命名分布过窄（8 schema × 18 base × 7 suffix 的笛卡尔小流形），模型在 OOD 上退化成背诵这个小流形"。真实表名（HF `CREATE TABLE` 收割，来自 gretelai/synthetic_text_to_sql + b-mc2/sql-create-context，Apache-2.0 / cc-by-4.0）命名多样、无统一 schema 前缀规律，无法被一个小流形记住。**替换**比"稀释"更能干净地检验"命名分布是不是根因"：若替换后泄漏仍在，则根因不止于命名分布。仍保留 10% 合成名，是为了让"逐字命中原合成池"这个招牌指标在 B1 上**仍可测**（否则合成名彻底消失，原指标恒 0，无法与基线同列对比）。

**Alternatives considered**：
- **纯稀释（保留 400 合成名 + 加大真实名）**：合成名占比下降但绝对量不变，模型仍可能优先记忆结构化的合成名（更"好记"）。作为对照不如替换干净——若要，可作 B1' 加做，但非门槛。
- **纯真实（0 合成名）**：招牌指标"逐字命中合成池"恒 0，无法与基线逐列对比；且无法回答"模型是否改背真实训练名"。**否决**——见决策 3，改背真实名恰是最有价值的观察，须可测。
- **程序化扩充合成生成器（加更多 schema/base 词）**：仍是合成流形、仍可被记住，不检验"真实分布"假设。**否决**。

**待用户确认的参数**：真实名目标量（默认 ≥2000）、合成名保留比（默认 10%）。默认值已足以检验假设；用户可调。

## 决策 2：B2 开放域弃权负样本的来源与配比

**Decision**：**合成蒸馏负样本**——新增一个 held-out 隔离的"弃权模板家族"，程序化生成**看似 ETL 但无字面读写表**的脚本，标签恒 `{"reads":[],"writes":[]}`：
- 纯计算/日志脚本（pandas 变换、`print`、`logging`、数值聚合，无 SQL、无 `saveAsTable`/`INSERT`）；
- 表名**仅出现在注释/被打印的字符串**里（正对 041 约定 A "注释/仅打印的 SQL 不算"）；
- 动态拼接表名（`f"{schema}.{tbl}"` 变量驱动，无字面名）。

配比：负样本占训练集 **20%**（`train_size` 不变，其中 20% 为空标签负样本，80% 为既有正样本）。其余配方逐字不变。

**Rationale**：041 的病是"OOD 上乱背/过度抽取"。弃权训练直接教"证据不足→输出空"。**用合成蒸馏而非真实空标签脚本**有三重理由：① 保持"合成-only 训练"范式纯净——掺真实脚本会引入"见过真实分布"的混杂变量，破坏 041 全篇"零真实标注"的立论；② 确定性可复现（固定 SEED，无第三方脚本依赖/发布顾虑）；③ 负样本恰好覆盖约定 A 的三类范围外情形（注释/打印、动态名、无表），让"弃权"与金标口径同源。20% 是弃权训练常见量级（够学会弃权、不至于压垮正样本 recall）。

**Alternatives considered**：
- **真实空标签脚本（realeval 采集里 80/139 空金标那批）**：破坏"合成-only"纯净性 + 与真实评测集同源有泄漏测试集之嫌（train/test 污染）。**否决**。
- **更高负样本比（如 50%）**：易过度弃权、recall 崩塌成"恒空"退化解。作为 trade-off 曲线的一点可选加做，非默认。
- **在 loss 上加显式弃权奖励/RL**：超出"同规模改训练数据"这一最小消融，引入新目标函数=多个变量，破坏消融纯净。**否决**（留 future work）。

**待用户确认的参数**：负样本占比（默认 20%）。

## 决策 3：泄漏度量如何对 B1/B2 保持诚实且可比（防刷指标）——**核心诚实性设计**

**Decision**：泛化 `leak_analysis.py`，泄漏检测**按"被测模型自己的训练表名池"评测**，而非硬编码回放原合成池。具体：
- 新增 `--train-pool <pool.json>` 参数；每个变体构造器 `data/antileak.py` 落盘一份 `pool.json`（该变体训练集实际用到的**全部表名**，即"该模型见过的名字全集"）。
- 泄漏 = 幻觉表名中**逐字命中该模型自有训练池**的比例（+ 合成形态率仍按原 `_synthetic_shaped` 判据，度量"像合成流形"）。
- **B1 同时报两列**：(i) 逐字命中**原合成池**（与基线同源，看合成名是否还被背）；(ii) 逐字命中**B1 自有训练池**（看模型是否改背真实训练名）。
- 默认（不传 `--train-pool`）回放原合成池 → **基线/3B/JVM 行为逐字不变，向后兼容**。

**Rationale**：若 B1 只换名字、泄漏检测仍只查原合成池，指标会**假性归零**（模型没见过合成名当然不吐）——这是刷指标、不诚实。真问题是"模型是否仍在 OOD 上背诵**它自己的**训练分布"。按自有池评测才 gold-无关且公平：
- B1 若"逐字命中自有池"仍高 → 命名分布不是根因，换成真实名照背 → **补救无效**，负结果更硬；
- B1 若两列都低 → 增广真的打散了可背诵的流形 → **补救部分有效**，诚实上报。
两种都靠这一个泛化才测得出。B2 同理按其自有池（含负样本）测，看弃权是否让幻觉/泄漏整体下降。

**Alternatives considered**：
- **一律只查原合成池**：B1 假性归零、不诚实。**否决**。
- **查"任意非金标非脚本名"= 全部幻觉**：那就是幻觉率本身，丢掉"背诵训练分布"这一机制判据。**否决**——泄漏的价值正在于区分"乱抽真实动态名"（大模型的错）与"背训练名"（小模型的病）。

## 决策 4：消融纯净性如何保证与核验

**Decision**：B1/B2 均复用 `train/sft_qlora.py`，**只改 `--data`**（指向变体 train.jsonl），`--base-model` 固定 `Qwen/Qwen2.5-Coder-1.5B-Instruct`，其余（SEED=20260703 / LoRA r16 α32 dropout0.05 / 2ep / max2048 / bf16 / 同 SYSTEM_PROMPT）一字不改。核验：单测断言变体 train.jsonl 与基线的差异只在"表名分布"（B1）或"含 20% 空标签样本"（B2）；训练命令行 diff 只有 `--data`/`--out`。

**Rationale**：spec FR-003 + SC-003 要求"恰好一个维度"。base/超参/prompt 任何漂移都会让"泄漏变化归因于命名分布/弃权"失效。锁死配方是消融的生命线。

**Alternatives considered**：为 B 调 LR/epoch 让数字更好看 → 直接破坏可归因性，**否决**。

## 决策 5：评测集与基线冻结

**Decision**：真实评测复用 `realeval/gold/real.jsonl`（n=139/非空 59）+ `eval_real.py`；合成 held-out 复用 `data/out/heldout.jsonl` + `eval/evaluate.py`；基线（1.5B run1/run3、3B）数字取 `paper-negative-result-findings.md` 冻结值，**不重跑**。JVM（`real-jvm.jsonl`）为可选加做，不作门槛。

**Rationale**：可比性要求单一金标单一评测器（spec FR-004）。基线已实测冻结，重跑只增噪声与算力。JVM 复评能加厚"补救跨语言也失败"，但主结论在 py/sh 上已足够决定性。

**Alternatives considered**：为 B 扩真实集 → 改变分母、破坏与基线可比。**否决**（扩集是另一独立工作）。

## 未决/交由用户的 plan 级旋钮（默认已可跑）

| 旋钮 | 默认 | 影响 |
|---|---|---|
| B1 真实名目标量 | ≥2000（HF limit 20000） | 太少不足以打散流形；默认足够 |
| B1 合成名保留比 | 10% | 保留是为招牌指标可测；0% 则原指标失效 |
| B2 负样本占比 | 20% | 太高→过度弃权 recall 崩；太低→学不会弃权 |
| JVM 复评 | 不做（可选） | 做则加厚跨语言结论 |

**所有默认值均可直接进入 `/speckit-tasks` 与实现**；上述四旋钮属实现细节，若用户无异议即采默认。
