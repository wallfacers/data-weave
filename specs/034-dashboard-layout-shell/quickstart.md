# Quickstart / 验证指南：Dashboard-01 外壳布局迁移（保留多 Tabs）

前置：后端 `:8000`、前端 `:4000` 已起（见 [README.md](../../README.md) / CLAUDE.md Build & Run）。本特性纯前端，后端可用默认或 H2 profile。

> Turbopack 全局 CSS 陈旧缓存坑：改动后样式不显时，先 `rm -rf frontend/.next` 重启 `pnpm dev`，别怀疑特异度（[[turbopack-global-css-stale-hmr]]）。

## 1. 类型/静态门

```bash
cd frontend && pnpm typecheck        # 零错误
pnpm design:lint                      # DESIGN.md 结构校验通过
```

## 2. 单元测试（契约 A/B 自动断言）

```bash
cd frontend && pnpm test              # 覆盖面包屑派生函数（A1/A2/A3）、tabLabel 复用一致性（B3）
```

预期：面包屑派生函数对「入口视图」「无分组视图」「带 params 详情视图」三类输入分别返回二/三/四级节点序列；`tabLabel()` 导出后在 `tab-bar.tsx` 与 `breadcrumb.tsx` 两处调用结果一致。

## 3. 浏览器验证门（契约 A4-A7、B1/B2、C1-C4）

登录后打开工作区，逐项目测：

### 3.1 面包屑基础行为（US1）

1. 依次打开「任务调度 > 运维总览」「系统设置」两个 Tab。
2. **验 A1**：运维总览面包屑显示「项目 > 运维监控 > 运维总览」三级，与左侧导航当前高亮分组一致。
3. **验 A2**：系统设置面包屑显示「项目 > 系统设置」二级（或按其归属分组展示），不留白级、不报错。
4. **验 A5**：点击左侧导航底部折叠按钮收起为 icon rail，确认面包屑内容不受影响、依旧完整可读。
5. **验 A6**：DevTools 检查面包屑行与 Tab 条、内容区之间无 `border-b`/`border-t` 元素。

### 3.2 多 Tabs 独立与零回退（US2）

1. 打开 5 个不同视图 Tab（如运维总览、指标概览、目录树、告警、设置）。
2. **验 A4**：随机顺序切换 Tab 至少 10 次，每次切换后面包屑与内容区同步对应当前 Tab，无残留/闪烁。
3. **验 A3**：从运维总览的某个实例列表点开一条实例日志（打开 `instance-log` 详情 Tab），确认该 Tab 的面包屑末级显示实例标识，且与该 Tab 在 Tab 条上的标签文案一致。
4. **验 B1**：对上述 Tab 分别执行新开/关闭/固定/取消固定操作，行为与改造前一致（无需对照旧版本，凭已知交互预期核对）。
5. **验 B2**：切到某个已加载数据并设置过筛选条件的 Tab（如指标概览开启自动刷新后切走再切回），确认筛选/开关状态未因本次改造重置。

### 3.3 卡片/表格栅格审计（US3）

逐一打开含统计卡片或数据表格的视图，对照标杆确认版式一致：

- 指标概览（`metrics-view.tsx`）—— 卡片栅格参照基准
- 集群/Fleet、报表等含卡片视图
- 系统设置、数据源、新鲜度、运维各面板等含 `DataTable` 视图

对每个视图：
- **验 C2**：统计卡片是否使用与指标概览一致的响应式栅格（列数随视口变化、间距/圆角/阴影一致）。
- **验 C3**：数据表格是否呈现统一边框 frame（与 033 分支交付的规范一致）。
- **验 C1**：确认页面中没有新出现的面积图/折线图等可视化图表。
- **验 C4**：无卡片/表格的视图（如画布类）未被强行插入空卡片容器。

### 3.4 双主题 + 窄视口

切换浅色/深色主题，面包屑文字/背景在两种主题下均清晰可辨；缩窄浏览器视口，确认面包屑文案截断/响应式收敛不撑破 header 行高度。

## 4. 通过标准

- typecheck / design:lint / test 全绿。
- 全部抽查视图面包屑三级（或二级兜底）正确、随 Tab 切换零残留（SC-001/SC-003）。
- 90% 以上视图卡片/表格版式与标杆一致（SC-002）。
- 无新增图表库依赖、无新增图表渲染（SC-004）。
- 多 Tabs 全部既有操作 100% 通过回归（SC-005）。

参照 [contracts/breadcrumb-visual-contract.md](contracts/breadcrumb-visual-contract.md) 逐条断言核对。
