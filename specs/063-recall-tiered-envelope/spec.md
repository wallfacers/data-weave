# Feature Specification: 召回回收 · 置信度分层复核信封

**Feature Branch**: `063-recall-tiered-envelope`

**Created**: 2026-07-11

**Status**: Draft

**Input**: User description: "召回回收：置信度分层复核信封接进血缘抽取 serving。免费确定性召回天花板 0.764，自动采纳层(治理阈默认0.95)可直接入库，复核候选层 surface 并集召回给人工复核队列。本轮只做 serving 侧分层产出 + 离线证明，不碰平台消费面。"

## 背景

自托管 3B 血缘抽取器（059）精度已超两个前沿 teacher（ALL-p 0.684 > deepseek-pro 0.593 > qwen-max 0.413），但**召回是唯一实质弱项**：语义 grounding 后召回 0.703，teacher 带 0.77–0.81。这是换精度付出的代价（模型训得保守）。

探针实测定界（gold C，61 非空脚本 / 148 真表）：免费确定性手段（模型 ∪ SQL-AST 通道）的召回天花板 = **0.764**，落在 teacher 带下沿甚至以下——teacher 的额外召回来自动态名/框架/配置驱动的表，不付费/不重训够不着。且裸并集会赔精度（掉到 0.685）。

**故本特性不追 teacher 的 0.81**，而是把召回从 0.703 往 0.764 天花板推，**用置信度分层守住自动入库精度**，产出**分层复核信封**：高置信自动采纳、其余进人工复核队列。这是 052/054 的置信度校准框架从未接进 059 serving 的落地。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 复核者拿到召回回收的候选队列 (Priority: P1)

数据治理复核者对一个脚本要人工确认血缘。今天抽取器只给一份保守的表清单（召回 0.703），漏掉的表复核者根本看不到、无从补。分层信封后，复核者额外拿到一层**复核候选**——把免费手段能 surface 的全部血缘（并集召回 0.764）按置信度降序摆出来，复核者从最可能对的先看、逐条拍板，不再漏。

**Why this priority**: 这是召回回收的**主交付**——把 +6pt 召回 surface 给人工，直接回应「换精度丢的召回」这一唯一弱项。单独实现即有价值：复核者少漏血缘。

**Independent Test**: 在 gold C 上离线跑分层信封，验证「自动采纳层 ∪ 复核候选层」覆盖的真表召回逼近 0.764（vs 模型独抽 0.703），且候选按置信度可排序。

**Acceptance Scenarios**:

1. **Given** 一个含模型漏抽、但 SQL-AST 通道能解析出的真表的脚本，**When** 抽取，**Then** 该真表出现在复核候选层（不在自动采纳层被静默丢弃）。
2. **Given** 一批非空脚本，**When** 抽取，**Then** 复核者可见候选按置信度（校准 precision）降序排列，最可能对的在前。
3. **Given** gold C 全集，**When** 离线评测，**Then** 自动层 ∪ 复核层召回 ≥ 0.76，显著高于模型独抽 0.703。

---

### User Story 2 - 自动入库只收高置信血缘 (Priority: P2)

平台把抽取结果自动写入血缘库（错血缘有治理后果）。分层信封把「治理安全」显式化：**自动采纳层**只含累计校准 precision ≥ 治理阈（默认 0.95）的候选，可直接入库；达不到阈的一律降级到复核层，绝不自动入库。

**Why this priority**: 保证召回回收**不以牺牲自动入库精度为代价**——自动层比今天平铺输出更准。P2 因为它依赖 P1 的分层机制，是分层的「另一半」价值。

**Independent Test**: 离线量自动采纳层在 held-out（CV 去偏）口径下的 precision，确认 ≥ 治理阈；确认低置信候选不出现在自动层。

**Acceptance Scenarios**:

1. **Given** 治理阈默认 0.95，**When** 抽取，**Then** 自动采纳层的 held-out precision ≥ 0.95。
2. **Given** 一个只有裸名、低置信通道命中的候选，**When** 抽取，**Then** 它进复核层而非自动层。
3. **Given** gold C 全集，**When** 离线评测，**Then** 自动采纳层 precision 不低于今天平铺输出（无回归）。

---

### User Story 3 - 运维可调治理阈并可回滚 (Priority: P3)

不同部署对治理容错度不同：有的要极严（宁可全进复核），有的可放宽换更多自动入库。运维可通过配置调自动采纳阈（默认 0.95 治理严格；文档标注 0.90 为统计稳定膝点，自动层召回约 0.45），并可一键完全关闭分层退回旧的单一输出。

**Why this priority**: 部署弹性与安全阀。P3 因为默认值已可用，可调/回滚是增强。

**Independent Test**: 改配置阈重跑，验证自动/复核层随阈迁移；关闭分层开关，验证响应退回旧的单一表清单语义。

**Acceptance Scenarios**:

1. **Given** 治理阈配置为 0.90，**When** 抽取，**Then** 自动采纳层扩大（纳入 agree 层）、复核负载下降。
2. **Given** 分层完全关闭，**When** 抽取，**Then** 响应等价于旧的单一 reads/writes 输出，旧字段语义不变。
3. **Given** 任意阈，**When** 抽取，**Then** 复核候选层始终 surface 全并集（召回天花板不随阈变化）。

---

### Edge Cases

- **脚本无血缘（空脚本）**：自动层与复核层均空，`tiered=false`，与旧弃权行为一致。
- **模型与 SQL 通道对同一表限定/裸名不一致**（`t` vs `db.t`）：canon 归一，同一表只计一次、只占一层，不因限定名重复计数。
- **SQL-AST 通道未开火**（纯框架/动态名脚本）：复核层 = 模型候选中未达自动阈者；召回天花板退化为模型独抽水平（诚实，不虚构候选）。
- **治理阈设到 0**：全并集进自动层、复核层空（等价旧并集口径）。
- **校准样本小（148 表）导致 ≥0.95 边界抖动**：以 CV 去偏后的 held-out precision 为准固化，披露边界不确定性，不用样本内乐观值。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 抽取结果 MUST 分为两层——**自动采纳层**（治理安全、可直接入库）与**复核候选层**（进人工队列，不自动入库）。
- **FR-002**: 候选来源 MUST 为「模型抽取表集 ∪ SQL-AST 通道表集」，二者在 canon 口径下去重合并为互斥候选，避免限定名/裸名重复计数。
- **FR-003**: 每个候选 MUST 带**置信度信号**（通道归属 × 名字限定性分级），并据其校准 precision 归入某一置信级。
- **FR-004**: 候选进入自动采纳层 MUST 满足其累计校准 precision ≥ **治理阈**（默认 0.95）；否则降级到复核候选层。
- **FR-005**: 校准 precision 常量 MUST 在**独立集（测试集 A 金标，与 gold C 按来源隔离）上固化冻结**，gold C 作为纯 held-out 评测集不参与定级；固化后供部署期使用，部署期 MUST NOT 依赖金标重算。禁止用 gold C 既定级又评测（避免循环）。
- **FR-006**: 复核候选层 MUST 按置信度降序排列，供复核者从最可能对的先看、按治理阈早停。
- **FR-007**: 治理阈 MUST 可通过配置调整；系统 MUST 支持一键完全关闭分层、退回旧的单一表清单输出（语义不变、向后兼容）。
- **FR-008**: 分层 MUST 为确定性后处理，同版本模型 + 同输入 + 同阈 → 同输出（不引入新的随机性/GPU 依赖）。
- **FR-009**: 交付 MUST 含离线评测证明——在 gold C 上量自动层 precision/recall、复核层召回、复核负载，并与两个前沿 teacher 同底对照，报告落盘可复核。
- **FR-010**: 方向（读/写）MUST 沿用既有 AST 锚定的方向修正，分层不改变方向判定；当同一表在模型与 SQL-AST 通道（agree 层）方向判定冲突时，MUST 以 **SQL-AST 的 AST target 锚定方向为准**。

### Key Entities *(include if feature involves data)*

- **候选血缘边（candidate）**：一个 (表名, 方向, 列) 单元，携带 `tier`（置信级：sql_qual/model_bare/agree/model_qual/sql_bare）与 `confidence`（该级 held-out 校准 precision）。
- **自动采纳层（auto-accept tier）**：累计校准 precision ≥ 治理阈的候选集合，可直接入血缘库。
- **复核候选层（review tier）**：并集中未进自动层的候选集合，进人工复核队列，按 confidence 降序。
- **置信级校准表（calibration table）**：每个置信级的 held-out 经验 precision 常量，CV 去偏固化，决定分级与阈切分。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 复核者可见的血缘召回（自动层 ∪ 复核层，gold C 非空脚本）≥ **0.76**，相对模型独抽 0.703 回收 ≥ +5pt。
- **SC-002**: 自动采纳层在 **gold C 纯 held-out**（分级常量固化于测试集 A、gold C 不参与定级）口径下 precision ≥ 治理阈（默认 0.95），且不低于今天平铺输出的精度（无回归）。
- **SC-003**: 复核候选层按置信度排序，复核者可在治理阈处早停；报告给出每脚本复核负载（候选/脚本）供人力评估。
- **SC-004**: 治理阈从 0.95 调到 0.90 时，自动采纳层召回从约 0.05 升到约 0.45（可调性可验证）；关闭分层时输出与旧单一清单逐字节等价。
- **SC-005**: 三方（自托管 3B / qwen-max / deepseek-pro）同施分层的离线对照报告落盘，数据第三者可复核；诚实披露 ≥0.95 治理严格下自动层召回过低（约 0.05）的代价，不为达标放松 CV 护栏或谎报精度。

## Assumptions

- **召回上限定界**：免费确定性手段（模型 ∪ SQL-AST）召回天花板 ≈ 0.764，追平 teacher 0.81 明确排除（需付费 teacher 或重训，均为非目标）。
- **范围切割**：本特性只做**抽取 serving 侧的分层产出 + 离线证明**。平台侧消费分层（复核队列 UI、自动层入库落表、复核结果回写）是**下一个特性**，本轮不碰后端/前端。
- **复用既有资产**：复用 059 的语义 grounding、dir_fix 方向修正、SQL-AST 通道（channel_router）、052/054 的置信度校准与 CV 去偏框架；不新增 AST/SQL 解析逻辑。
- **数据环境**：分级校准常量固化于**测试集 A（金标，052/054 已在其上校准+CV，可复用）**；**gold C（153 条 GitHub-fresh，与训练银标及测试集 A 按来源隔离 + 去污染）为纯 held-out 评测集**，不参与定级。测试集 A 校准样本小（141 表），沿用 052/054 的 CV 去偏并披露边界抖动。
- **无新资源**：不新增 GPU、不付费 teacher 调用、不新增人工标注。
