# Schema DDL Contract: 051 就绪态物化

单一权威 DDL：`backend/dataweave-api/src/main/resources/schema.sql`。无迁移脚本。schema_version **0.9.0 → 0.10.0**（三处同步：DB 行 / 文件头注释 / 项目版本）。H2 + PG 双兼容（H2 in-memory profile 与 PG 默认都要跑通）。

## 变更清单

### C1 — `task_instance` 加列

```sql
-- 加在 task_instance 表定义内，version 列后：
unmet_deps           INTEGER NOT NULL DEFAULT 0   -- 051：未满足依赖数（上游+跨周期），0=就绪；见 data-model.md
```

### C2 — 认领热索引扩列

```sql
-- 替换现有 idx_task_instance_claim：
--   旧：(state, run_mode, deleted, updated_at)
--   新：(state, run_mode, deleted, unmet_deps, updated_at)
CREATE INDEX IF NOT EXISTS idx_task_instance_claim
  ON task_instance (state, run_mode, deleted, unmet_deps, updated_at);
```

### C3 — 新表 `readiness_signal`（事务型 outbox）

```sql
CREATE TABLE readiness_signal (
    id                   BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    tenant_id            BIGINT NOT NULL,
    project_id           BIGINT NOT NULL,
    kind                 VARCHAR(16) NOT NULL,          -- TERMINAL | RESET
    upstream_instance_id UUID NOT NULL,
    workflow_id          BIGINT,
    workflow_instance_id UUID,
    workflow_node_id     BIGINT,
    biz_date             VARCHAR(32),
    processed            SMALLINT NOT NULL DEFAULT 0,
    created_at           TIMESTAMP,
    processed_at         TIMESTAMP
);
CREATE INDEX IF NOT EXISTS idx_readiness_signal_pending ON readiness_signal (processed, id);
```

```sql
INSERT INTO schema_version (version, applied_at, description)
VALUES ('0.10.0', CURRENT_TIMESTAMP, '051 就绪态物化：task_instance +unmet_deps + idx_task_instance_claim 扩列 + readiness_signal outbox 表');
```

同步文件头注释追加一行 `→ 051 就绪态物化=0.10.0（…）`，`DROP TABLE IF EXISTS readiness_signal;` 加入顶部 DROP 段。

## 双方言注意（复用既有约定）

- `GENERATED BY DEFAULT AS IDENTITY`：与既有表（workflow_node/edge）同款自增，H2/PG 均支持；取自增主键用 `GeneratedKeyHolder`（勿用 H2 旧 `CALL IDENTITY()`，见既有 alert 教训）。
- 索引非 partial（H2 兼容）。
- `SKIP LOCKED` 领取：PG 原生支持；H2 需确认版本支持 `FOR UPDATE SKIP LOCKED`（既有 claim 已用 SKIP LOCKED，沿用同款，H2 测试验证）。
- 时间戳 `CURRENT_TIMESTAMP`；字符串拼接场景用 `CONCAT`（勿用 `||`）。

## 契约测试（contract-level）

- `task_instance.unmet_deps` 列存在、NOT NULL、DEFAULT 0，H2/PG 均建表成功。
- `idx_task_instance_claim` 含 unmet_deps；`readiness_signal` 表 + `idx_readiness_signal_pending` 建成。
- `schema_version` 最新行 = `0.10.0`，与文件头一致。
- 认领查询 `WHERE state='WAITING' AND run_mode=? AND deleted=0 AND unmet_deps=0 ... FOR UPDATE SKIP LOCKED` 在 H2/PG 均可执行。
- `readiness_signal` 的 `FOR UPDATE SKIP LOCKED` 领取在 H2/PG 均可执行。
