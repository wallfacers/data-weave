# Data Model: 认领就绪态物化 + 性能链收口

schema_version: **0.9.0 → 0.10.0**（MINOR：新表 `readiness_signal` + `task_instance` 加列 + 索引扩列）。schema.sql 是单一权威 DDL，无迁移脚本；DB 行 / 文件头 / 项目版本三处同步升 0.10.0。

## 1. `task_instance` 加列 `unmet_deps`

```sql
-- 就绪态物化（051）：该 WAITING 实例尚未满足的依赖数（DAG 内上游 + 适用跨周期），0=就绪可认领。
-- 由 ReadinessInitializer 物化时算初值；ReadinessMaintainer 按权威状态 scoped 重算维护；ReadinessReconciler 兜底对账。
unmet_deps           INTEGER NOT NULL DEFAULT 0
```

- **位置**：`task_instance` 表内（`version` 列后追加）。
- **语义**：`unmet_deps=0` ⇔ 该实例所有 DAG 内上游门 + 适用跨周期门均已满足，可进入认领。`>0` ⇔ 仍有未满足依赖。
- **生命周期**：
  - 物化（`WorkflowTriggerService`）：`ReadinessInitializer` 算 `= 上游 edge 数 + 适用跨周期依赖数（非首周期豁免）`；无依赖 → 0。
  - 维护（异步）：上游/跨周期满足方终态 → 信号 → `ReadinessMaintainer` 对受影响下游 scoped 重算 → CAS `SET unmet_deps=<重算值>`。
  - 回退（FR-007）：上游被 rerun/reset → RESET 信号 → 重算把 unmet 加回（权威态覆盖）。
  - 对账：`ReadinessReconciler` 低频重算自愈漂移。
- **不变量**：非负；由权威重算落值（非盲递减），并发 CAS 收敛同值。

DROP INDEX ... idx_task_instance_claim;   -- 概念上：旧 (state, run_mode, deleted, updated_at)

- `idx_task_instance_node_bizdate (workflow_node_id, biz_date, deleted)` 保留——Maintainer/Reconciler 重算跨周期时按 (node, bizDate) 查上周期 SUCCESS 仍需它。

## 2. 新表 `readiness_signal`（事务型 outbox）

```sql
CREATE TABLE readiness_signal (
    id                   BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    tenant_id            BIGINT NOT NULL,
    project_id           BIGINT NOT NULL,
    kind                 VARCHAR(16) NOT NULL,   -- TERMINAL(上游/跨周期满足方到终态) | RESET(上游 rerun/reset 回退)
    upstream_instance_id UUID NOT NULL,          -- 触发信号的实例（完成或被重置的那个）
    workflow_id          BIGINT,                 -- 快照：满足方 workflow_id（跨周期反查用）
    workflow_instance_id UUID,                   -- 快照：满足方 wf-instance（同 DAG 后继定位用）
    workflow_node_id     BIGINT,                 -- 快照：满足方 node（edge/dependency 反查的 from/depend 端）
    biz_date             VARCHAR(32),            -- 快照：满足方 bizDate（跨周期逆偏移用）
    processed            SMALLINT NOT NULL DEFAULT 0,  -- 0=待处理 1=已处理
    created_at           TIMESTAMP,
    processed_at         TIMESTAMP
);
-- Maintainer 轮询：WHERE processed=0 ORDER BY id ... FOR UPDATE SKIP LOCKED（多 master 各领各的）。
CREATE INDEX IF NOT EXISTS idx_readiness_signal_pending ON readiness_signal (processed, id);
```

- **写入**：`ReadinessSignalWriter` 在完成/reset **事务内** INSERT 一行（与 `casTaskTerminal` 同事务）→ no-loss 构造保证。单行，不做扇出写。
- **消费**：`ReadinessMaintainer` 批量 `FOR UPDATE SKIP LOCKED` 领取 `processed=0` → 解析下游 D → 重算 → `processed=1, processed_at=now`。
- **幂等**：即便信号被重复处理（崩溃重放/多 master），下游 scoped 重算读权威态 → 同结果，安全。
- **保留/清理**：`processed=1` 行可由既有历史清理机制或独立 TTL 清理（tasks 阶段定；非正确性关键，仅存储回收）。

## 4. 状态语义速查

- **就绪判定**：`state=WAITING AND unmet_deps=0`（+ 既有 `backfill_held`/wf-instance 门保留在 selectRunnable）。
- **上游放行**：STRONG（edge.strength=STRONG/NULL）= pred `SUCCESS`；WEAK = pred 任意终态（`SUCCESS`/`FAILED`）。
- **跨周期放行**：仅 CRON 实例检查；上周期实例（node, 逆偏移 bizDate）存在 `SUCCESS`；`biz_date < earliest_biz_date` 首周期豁免；非 CRON/缺字段直通。
- **重算值**：`unmet_deps = (未满足上游 edge 数) + (未满足适用跨周期依赖数)`；两者均 0 → 就绪。**物化初值同此权威口径**（只计未满足，非依赖总数——见 FR-002 / contracts ReadinessInitializer / C1）。
- **跨周期逆偏移**：反查后继时求 `B'` 使 `offsetBizDate(B', offset)==B`；不假定 offset 可逆——用正向枚举候选 bizDate 匹配（或证明 offset 简单可逆再直算），覆盖月末/工作日等非平凡 offset（U1）。
