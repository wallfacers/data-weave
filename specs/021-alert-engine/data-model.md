# Phase 1 Data Model: 告警引擎

权威 DDL 真相源 = `backend/dataweave-api/src/main/resources/schema.sql`。本文为设计视图(字段/约束/索引/状态机),实现时落 schema.sql 并升 `schema_version`。**双方言**:PG/H2 通用 DDL(`GENERATED BY DEFAULT AS IDENTITY`、`UUID`、JSON 存 `VARCHAR`、枚举 `VARCHAR(32)`、`IF NOT EXISTS` 不用、DROP 逆序加);拼接用 `CONCAT` 不用 `||`;布尔/标志用 `SMALLINT`。

**部分唯一**:活跃(FIRING/ACKED)同 fingerprint 只一条 → 应用层保证(查活跃 + CAS),H2/PG 部分索引差异故不用 DB 部分唯一索引,用 `(tenant_id, fingerprint, state)` 普通索引 + 应用 CAS。
索引:`(tenant_id, state)`、`(tenant_id, rule_id)`、`(tenant_id, fingerprint)`。

### 状态机

```
            信号满足(新 fingerprint)
   ┌──────────────────────────────────────► FIRING
   │                                          │  │  │
   │  抑制窗口内同 fingerprint 再满足          │  │  └─► 命中 silence ─► SUPPRESSED
   │  → count++ / last_fired_at(不分发,自环)  │  └─► 人工 ACK ─► ACKED(窗口内不再分发)
   │                                          └─► auto_resolve 且条件清除 ─► RESOLVED(发恢复通知)
```
不变量:状态转换幂等(CAS `WHERE state=?`);SUPPRESSED 优先级最高;RESOLVED/ACKED 后同 fingerprint 新触发可再建 FIRING。

## 关系图

```
alert_rule 1──N alert_event 1──N alert_notification N──1 alert_channel
alert_rule N──路由匹配──N alert_channel  (经 alert_route.match_json/channel_ids)
alert_silence ──label 匹配──► 抑制 alert_event(标 SUPPRESSED)
alert_poll_fire ──(rule_id,poll_slot) guard──► MetricPollEvaluator 单点
```
