# Feature Specification: Lineage Graph Explorer（血缘图探索器）

**Feature Branch**: `052-lineage-graph-explorer`

**Created**: 2026-07-07

**Status**: Draft

**Input**: User description: "当前血缘模块的图展示问题，我看起就一个 Demo 没什么区别，你看下大家在血缘领域是如何展示的，并且结合数据中台的场景，把这个展示给我，血缘图的查询操作完善掉，方便后续的 vibecoding 数据中台的时候，前端页面的改造，你来思考"

## 背景与问题（Context）

现有血缘视图（`lineage-view`）用一张**手绘 SVG 网格**渲染图：节点按数组下标塞进 4 列固定格子，边是正交折线；没有布局引擎、没有缩放/平移/自适应视野、没有小地图，跨页的边会被静默丢弃。交互上只支持**下游单向**、深度写死（10/20）、点节点即整图重载替换、没有图内搜索/过滤，影响分析只是一个扁平列表。后端已有的能力（上游、列级血缘、指标血缘、置信度/加工类型/来源渠道/人工修正状态、synced rows）**几乎都没在图上呈现**；而后端查询侧本身也有硬缺口：没有**按名字找节点**的搜索、`/graph` 全局接口是空桩、影响分析不返回边（无法高亮路径）、双向邻域接口丢边、节点不带 owner/新鲜度等展示属性。

结果就是"看起来像个 Demo"。本特性把血缘从演示级提升为**数据中台可用的血缘探索器**：既补齐图查询操作（搜索、双向、可控深度、原地展开、路径高亮、影响半径计数、按属性过滤），也把已有的丰富元数据用**业界通行的血缘图范式**呈现出来，为后续数据中台前端改造打好可复用的地基。

**业界血缘展示范式参考**（用于对齐目标形态，非实现约束）：以数据资产为节点、左上游→右下游的有向流；点击节点原地展开相邻一跳；表节点可展开成列清单做**列级血缘**；边用样式编码可信度/加工类型；提供搜索定位、聚焦某节点、影响/血缘半径统计、路径高亮、图例与节点详情面板。

## Clarifications

### Session 2026-07-07

- Q: 血缘探索器的页面外壳布局采用哪种？ → A: **三栏布局**（左 catalog 目录树 · 中血缘画布 · 右详情），但详情**不作常驻固定列**，改为**嵌入画布的可关闭面板**——与工作流实例 DAG 查看器一致（点选节点/边打开、带显式关闭按钮、宽度可调），复用其 `DetailPanelShell` + `renderSidePanel` 侧面板骨架。
- Q: 界面原语（图、树、输入框、下拉框、滚动、加载、刷新）是否必须符合项目规范组件？ → A: 是，**强制 reuse-first**——图渲染复用工作流 DAG 所用的图引擎（`@xyflow/react`），滚动→`DwScroll`、下拉→`DropdownSelect`、加载→`LoadingState`、刷新→`ViewRefreshControl`、卡片→`Card`+`--card-spacing`、图标→hugeicons；目录树/方向段控/深度步进器等若目录无对应原语，须新建并回填 DESIGN.md 组件目录。禁止手写同类原语（含手绘 SVG 图）。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 真实图引擎的双向血缘探索（Priority: P1）

数据工程师打开某张表或某列的血缘，看到一张**左上游→右下游自动布局**的有向图（而非固定网格）：可自由缩放、平移、一键自适应视野；可切换查看**上游 / 下游 / 双向**；可调节展开深度；点击任意节点即在**原地增量展开**其相邻节点（而非整图重载替换），并可再次收起。选中节点时高亮其直接连边，并在**嵌入画布的可关闭详情面板**中显示该节点详情。

**Why this priority**: 这是"从 Demo 到可用"的核心杠杆。没有真实布局与双向/原地展开，其余一切增强都建立在一张看不清的图上。单独交付即可让血缘从"能看一眼"变成"能真正顺着数据流查问题"。

**Independent Test**: 以一个已知有多级上下游的表为锚点打开血缘；验证图按上游左/下游右分层布局、无固定 4 列网格；缩放/平移/自适应视野可用；切换上游/下游/双向后图内容与边随之变化；点击一个下游节点后其下一跳被追加进当前图且原有节点保留，再次点击可收起。

**Acceptance Scenarios**:

1. **Given** 一张有 3 级下游、2 级上游的表，**When** 用户以该表为锚点打开血缘并选择"双向"，**Then** 图以锚点为中心、上游在左下游在右分层展开，所有连边完整呈现（不因分页丢边）。
2. **Given** 已渲染的血缘图，**When** 用户滚轮缩放并拖拽平移后点击"自适应视野"，**Then** 全图缩放平移到完整可见且居中。
3. **Given** 当前图只展示到 1 跳，**When** 用户点击某个下游节点的"展开"，**Then** 该节点的下一跳邻居被**追加**进现图、已有节点与视图位置保留，且该节点标记为已展开；再次点击则收起这些新增邻居。
4. **Given** 用户把方向从"下游"切到"上游"，**When** 切换发生，**Then** 图重新以锚点为根按上游布局，方向状态在视图上可见。
5. **Given** 选中某节点，**When** 选中态生效，**Then** 与其直接相连的边被高亮、嵌入画布的可关闭详情面板展示其类型/层/所属任务等属性。

### User Story 2 - 图内搜索与定位（Priority: P2）

用户不需要预先知道节点 id，只要在血缘视图内输入表名/列名/指标名的关键字，就能**搜索匹配的数据资产**、从结果中选一个，图便以它为锚点加载并**聚焦居中**到该节点。

**Why this priority**: 现状必须先知道节点 id 才能查血缘（无按名搜索接口），这是探索器的入口瓶颈。有了搜索，血缘图才从"给定起点看图"变成"想查哪个查哪个"，是数据中台日常排障的起点。

**Independent Test**: 在搜索框输入一个已存在表名的片段；验证返回带类型/层标注的候选列表；选择一项后图以该节点为锚点加载并在视野内居中高亮。

**Acceptance Scenarios**:

1. **Given** 项目内存在名为 `dwd_order_detail` 的表，**When** 用户在血缘搜索框输入 `order_detail`，**Then** 候选列表返回该表（含类型=TABLE、层=DWD 等标注），限定在当前项目范围内。
2. **Given** 搜索结果列表，**When** 用户点选其中一项，**Then** 图以该节点为锚点重新加载并把该节点居中、高亮为当前锚点。
3. **Given** 关键字无任何匹配，**When** 搜索完成，**Then** 展示明确的空态提示（非加载态、非报错）。

---

### User Story 3 - 影响分析与路径高亮（Priority: P2）

用户对一个节点做**影响分析（下游 blast radius）**：图上高亮出全部受影响的下游节点**与连接它们的边（完整路径）**，并给出"受影响 N 个下游资产"的**计数徽标**；反向也能看某节点的**全部上游来源路径**。用户还能在两个节点间**高亮所有连接路径**，快速回答"这张报表的数据到底从哪来 / 改这张表会炸到谁"。

**Why this priority**: 影响分析是血缘在数据中台最高频的业务价值（变更评估、故障溯源、下线评估）。现状影响分析只是扁平列表且**不返回边**，无法在图上高亮路径，价值大打折扣。

**Independent Test**: 对一个有多级下游的节点触发影响分析；验证图上受影响节点与其间的边被整体高亮成一条条可辨识的路径，并显示受影响下游总数徽标（区别于当前分页条数）。

**Acceptance Scenarios**:

1. **Given** 一个有 5 个可达下游资产的表，**When** 用户触发"影响分析"，**Then** 这 5 个节点及连接它们的边在图上被高亮为受影响路径，并显示"受影响 5 个下游资产"计数（反映真实可达总数，而非当前页节点数）。
2. **Given** 影响集大于展示上限，**When** 结果被截断，**Then** 图上给出截断提示并说明已达上限，计数仍反映真实可达总数（若可得）。
3. **Given** 用户选定锚点 A 与目标 B，**When** 触发"高亮 A→B 路径"，**Then** A 到 B 之间的所有连接路径被高亮，无路径时给出明确提示。

---

### User Story 4 - 列级血缘下钻与边语义编码（Priority: P3）

用户可把一个**表节点展开为其列清单**，在**列与列之间**查看列级血缘（派生关系），并看到每条边的语义编码：**可信度**（已确认/未验证/冲突/声明）、**加工类型**（直传/表达式/聚合）、**来源渠道**（SQL 解析/Agent/表单/脚本…）用不同边样式区分，配合**图例**和边悬浮/详情说明；表级与列级之间可切换粒度。

**Why this priority**: 列级血缘 + 边语义是血缘从"表关系图"升级为"可解释数据流"的关键，也是数据中台做字段级影响与合规溯源的基础。后端模型已带这些字段，缺的是图上的呈现；相对 P1/P2 属增强，故 P3。

**Independent Test**: 展开一个有列级派生关系的表节点，验证其列清单可见、列间派生边被绘制、不同可信度/加工/来源的边样式不同且有图例说明。

**Acceptance Scenarios**:

1. **Given** 一张带列级血缘的表，**When** 用户在表节点上"展开列"，**Then** 该表的列以可读清单呈现，列与其上下游列之间的派生边被绘制。
2. **Given** 图中存在不同可信度/来源的边，**When** 图渲染，**Then** 已确认/未验证/冲突、SQL 解析/推断等以可区分的边样式呈现，且提供图例解释每种样式。
3. **Given** 用户在表级与列级粒度间切换，**When** 切换发生，**Then** 图相应地在表节点视图与列级视图间转换且锚点保持。

---

### User Story 5 - 节点富信息与视图可分享（Priority: P3）

血缘节点直接携带**展示属性**：所属层（ODS/DWD/DWS/ADS）、负责任务、数据新鲜度 / 今日 synced rows、（若有）owner/标签，在节点上或详情面板可读；用户可把当前**聚焦的血缘视图导出**（图片或数据）或生成**深链**，粘给同事打开即定位到同一锚点与视图状态。

**Why this priority**: 富节点信息让血缘图承载运维决策所需上下文（谁产出、多新、影响多大）；导出/深链让血缘结论可协作、可嵌入排障流程。属锦上添花的增强，P3。

**Independent Test**: 打开任一表节点详情，验证展示层/任务/新鲜度等属性；复制深链在新会话打开，验证定位到同一锚点节点与方向/深度状态。

**Acceptance Scenarios**:

1. **Given** 一张有产出任务且当日有 synced rows 的表，**When** 用户查看其节点详情，**Then** 显示所属层、产出任务、新鲜度/当日同步行数等属性。
2. **Given** 当前聚焦某锚点、方向=上游、深度=3 的视图，**When** 用户复制深链并在新会话打开，**Then** 视图恢复到同一锚点、方向与深度。
3. **Given** 当前视图，**When** 用户导出，**Then** 得到当前血缘子图的可分享产物（图片或结构化数据）。

---

### Edge Cases

- **血缘存储不可用**：Neo4j 不可用时，视图给出明确的"血缘不可用"降级提示，而非空白或崩溃。
- **超大图 / 超上限**：可达节点超过后端上限时必须显式截断并提示"已达上限、结果不完整"，绝不静默截断当作"全量"。
- **孤立节点**：锚点无任何上下游时，图仅显示锚点自身并提示"无血缘关系"。
- **跨项目隔离**：搜索与图查询严格限定在当前项目范围内，不得泄漏他项目资产。
- **同名资产**：搜索命中多个同名/近名资产时，候选需带足够区分标注（层/类型/数据源）以消歧。
- **环 / 重复展开**：存在环或重复展开同一节点时，图不得无限增殖或重复添加同一节点/边。
- **已被人工移除的边**：被标记 REMOVED 的边不出现在图上；被 CONFIRMED 的人工修正需正确反映其可信度样式。
- **权限**：只读用户可浏览/搜索/影响分析；人工修正边等写操作仍走既有审批闸门与项目角色校验。

## Requirements *(mandatory)*

### Functional Requirements

**图渲染与交互（对应 US1）**
- **FR-001**: 血缘图 MUST 采用有向分层布局（上游一侧、下游一侧、锚点居中），不得再使用固定列/下标网格布局。
- **FR-002**: 用户 MUST 能对血缘图进行缩放、平移，并一键将全图自适应到可见视野。
- **FR-003**: 用户 MUST 能选择查看某锚点的上游、下游或双向血缘，且当前方向在视图上可见。
- **FR-004**: 用户 MUST 能调节血缘展开深度（在系统允许的上限内），深度当前值对用户可见。
- **FR-005**: 用户 MUST 能点击图中节点以**原地增量展开**其相邻一跳邻居（保留现有节点与视图状态），并能再次收起该展开。
- **FR-006**: 选中节点时系统 MUST 高亮其直接连边并在嵌入画布的可关闭详情面板展示该节点属性。
- **FR-007**: 系统 MUST 返回并绘制当前视图内所有相关连边，不得因分页/端点不在当前页而静默丢边。

**搜索与定位（对应 US2）**
- **FR-008**: 用户 MUST 能按名称关键字搜索数据资产（表/列/指标），无需预先知道其 id；结果限定当前项目范围。
- **FR-009**: 搜索结果 MUST 携带足以消歧的标注（类型、层、所属数据源等）。
- **FR-010**: 用户从搜索结果选定一项后，图 MUST 以该资产为锚点加载并将其在视野内居中、标为当前锚点。
- **FR-011**: 搜索无匹配时系统 MUST 展示明确空态（区别于加载态与错误态）。

**影响分析与路径（对应 US3）**
- **FR-012**: 用户 MUST 能对某节点做下游影响分析，系统 MUST 在图上高亮全部受影响下游节点**及连接它们的边**（完整路径），并 MUST 支持反向查看上游来源路径。
- **FR-013**: 影响分析 MUST 给出受影响下游资产的**真实可达总数**（与当前展示分页条数区分开）；当无法在上限内取得精确总数时须显式说明为下限/已截断。
- **FR-014**: 用户 MUST 能在两个指定节点间高亮所有连接路径；不存在路径时给出明确提示。

**列级血缘与边语义（对应 US4）**
- **FR-015**: 用户 MUST 能将表节点展开为其列清单，并查看列与列之间的列级血缘边。
- **FR-016**: 系统 MUST 以可区分的视觉样式编码边的可信度、加工类型与来源渠道，并 MUST 提供图例解释每种样式含义。
- **FR-017**: 用户 MUST 能在表级与列级粒度间切换，切换后锚点保持不变。
- **FR-018**: 被人工标记为移除的边 MUST NOT 出现在图上；人工确认的边 MUST 以对应可信度样式呈现。

**节点富信息与分享（对应 US5）**
- **FR-019**: 血缘节点/详情 MUST 能展示所属层、产出任务、数据新鲜度或当日 synced rows 等运维上下文属性（有则显示）。
- **FR-020**: 用户 MUST 能导出当前聚焦血缘子图为可分享产物（图片或结构化数据）。
- **FR-021**: 用户 MUST 能生成/复制当前视图的深链，重新打开后恢复同一锚点、方向与深度状态。

**通用 / 非功能**
- **FR-022**: 所有血缘搜索与图查询 MUST 严格遵守当前项目隔离，不得返回其他项目的数据资产或边。
- **FR-023**: 血缘存储不可用时系统 MUST 给出明确降级提示，不得空白或崩溃。
- **FR-024**: 结果达到系统上限时 MUST 显式提示截断且结果不完整，绝不将截断结果呈现为全量。
- **FR-025**: 图查询 MUST 能处理环与重复展开而不产生重复节点/边或无限增殖。
- **FR-026**: 现有的人工修正边（confirm/remove/revoke）能力 MUST 在新探索器中保留，且写操作 MUST 继续走既有审批闸门与项目角色校验。
- **FR-027**: 血缘查询与展示的能力 MUST 以可被前端页面复用的方式提供（服务于后续数据中台前端改造），使同一锚点/方向/深度/过滤的查询产生一致结果。

**布局与组件规范（对应本次澄清）**
- **FR-028**: 血缘探索器页面 MUST 采用三栏布局：左侧数据资产目录树、中间血缘画布、右侧详情区。
- **FR-029**: 节点/边详情 MUST 以**嵌入画布的可关闭面板**呈现，行为与工作流实例 DAG 查看器一致（点选节点/边打开、带显式关闭按钮、宽度可调），而非常驻不可关闭的固定列。
- **FR-030**: 所有界面原语（图渲染、目录树、搜索输入框、下拉框、滚动区、加载态、刷新入口、卡片容器、图标）MUST 复用项目既有规范 UI 组件与图渲染栈，不得手写同类原语（含手绘 SVG 图）；目录确无对应能力时须新建规范组件并回填组件目录。

### Key Entities *(include if feature involves data)*

- **数据资产节点（Asset Node）**：血缘图的顶点。代表数据源 / 表 / 列 / 指标之一；关键属性：类型、名称、所属层、（表/列）所属父级、展示属性（产出任务、新鲜度、synced rows、owner/标签等）。
- **血缘边（Lineage Edge）**：两个资产间的有向数据流。关键属性：方向、粒度（表级/列级）、可信度（已确认/未验证/冲突/声明）、加工类型（直传/表达式/聚合，列级）、来源渠道（SQL 解析/Agent/表单/脚本…）、关联任务、人工修正状态、模型版本。
- **血缘子图（Lineage Subgraph）**：一次查询的结果集合。关键属性：锚点、方向、深度、节点集、边集、是否截断及截断位置。
- **影响结果（Impact Result）**：某锚点的可达下游（或上游）闭包。关键属性：根节点、受影响节点集、连接边集（用于路径高亮）、真实可达总数、是否截断。
- **搜索候选（Search Candidate）**：名称检索命中的资产。关键属性：id、名称、类型、层、所属数据源等消歧标注。
- **视图状态（View State）**：可分享/深链的探索上下文。关键属性：锚点、方向、深度、粒度、过滤条件。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户仅凭资产名称片段即可在血缘视图内定位到目标节点并加载其血缘，无需事先获取任何内部 id；对已存在资产的名称搜索命中率为 100%。
- **SC-002**: 对任一给定锚点，用户能在同一张图内查看上游、下游与双向血缘，且当前视图内所有相关连边 100% 呈现（无因分页丢边）。
- **SC-003**: 对一个具有多级下游的节点做影响分析，图上高亮出受影响节点**及其连接边（路径）**，并显示与真实可达总数一致的计数；相较现状（扁平列表、无边、计数=当页条数）为可验证的改进。
- **SC-004**: 用户点击节点可原地增量展开/收起邻居，展开操作保留既有节点与视图位置（不整图重载替换）。
- **SC-005**: 血缘图支持缩放、平移与一键自适应视野；在展示达到系统节点上限的规模时仍可交互浏览而不失响应。
- **SC-006**: 边的可信度/加工类型/来源以可区分样式呈现且配有图例，用户无需打开详情面板即可从图上分辨"已确认 vs 推断"边。
- **SC-007**: 表节点可展开查看列级血缘，列到列的派生关系在图上可见。
- **SC-008**: 复制血缘视图深链在新会话打开后，恢复到同一锚点、方向与深度；导出可产出当前子图的可分享产物。
- **SC-009**: 所有搜索与图查询结果 100% 限定在当前项目范围内，跨项目零泄漏。
- **SC-010**: 结果达上限时 100% 给出显式截断提示；血缘存储不可用时给出明确降级提示，无空白/崩溃。

## Assumptions

- **范围为血缘的查询与展示（读侧 + 探索交互）**：血缘的**采集/写入**（SQL 解析、列级 catalog 播种 024、synced rows 025、脚本抽取 041）沿用现有实现，不在本特性重做；本特性消费其产出的图数据。
- **复用既有图渲染栈**：前端已引入并在工作流 DAG 中使用的图渲染库可复用于血缘图，无需引入新的重量级图框架（技术选型留待 plan 阶段确认）。
- **Neo4j 为唯一血缘存储**：延续现状，PG 血缘表已退休；所有图查询走图存储。
- **项目隔离既有机制沿用**：搜索/查询的租户与项目过滤复用现有 `TenantContext` + 项目范围校验。
- **人工修正闸门沿用**：边的 confirm/remove/revoke 等写操作继续走既有 PolicyEngine 审批闸门与 `project:manage` 角色校验，本特性不改闸门策略。
- **规模目标对齐现有上限**：以后端既有节点上限（约 2000）作为单视图展示的规模上界；超出以截断 + 分页/展开应对，不追求一次性渲染无界大图。
- **"数据中台场景"= 内部数据工程/运维用户**：目标用户为数据工程师、数仓/中台运维，具备血缘领域基本认知；非对外终端用户产品。
- **深链/导出为会话内可用**：深链在同一部署内定位锚点与视图状态；导出产物为图片或结构化数据（具体格式留待 plan）。
- **本特性主要驱动前端页面改造 + 少量后端查询接口补齐**：识别到的后端缺口（按名搜索、影响分析返回边、双向邻域带边、真实可达计数、节点富属性、路径高亮）作为支撑 US 的必要查询能力纳入范围。
