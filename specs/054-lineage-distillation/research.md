# Research: 自训小模型血缘蒸馏到生产可用（Phase 0）

spec 经 clarify 已消解全部 `[NEEDS CLARIFICATION]`；本文档固化关键技术决策（决策/理由/被否备选），供 tasks 与实现引用。所有数字锚定 041-R/047/050 已实测结论。

## R1 · 蒸馏范式：序列级硬标签蒸馏

- **决策**：序列级知识蒸馏——teacher 输出 JSON 血缘作为**硬标签**，学生按 SFT 学习。**不**做 logit 蒸馏。
- **理由**：teacher 是 API（DashScope / 阿里云 Anthropic 兼容），拿不到 logits；硬标签 SFT 是标准且够用的域内蒸馏形式，直接复用 `train/sft_qlora.py`。根因是分布错位而非软标签缺失，硬标签已能治本。
- **被否**：logit/白盒蒸馏（无 teacher logits，不可行）；RLHF/DPO（过重，无偏好数据）。

## R2 · 银标口径：交集为主 + 分歧字面门救回（clarify 定）

- **决策**：m1∩m2 一致的表直收；分歧表名仅当脚本内**字面出现**且方向可 AST 判定时救回；所有入选过字面子串硬门；方向优先 AST，不可解析取 teacher 方向，方向分歧且不可 AST 定则弃该边。
- **理由**：真实 ETL 稀疏 + teacher 高分歧（041-R 实测 18/48 分歧），严格交集会饿死非空银标量；字面门是最强精度保底（表名字面出现且被 teacher 标 → 极可能真）。m1-qwen 方向 0.87 是最强方向 teacher，但幻觉 0.30 高——由字面门滤，方向借 AST/teacher。
- **被否**：严格交集+字面门（精度最高但量塌，靠扩采补量成本高）；并集+字面门（量最大但噪声偏多，蒸馏噪声传导风险）。二者列为升级路径可切换项。

## R3 · 反泄漏：构造性保证 + 自有池审计

- **决策**：训练集**零合成表名**（纯真实银标），泄漏物理消除；仍用 `leak_analysis.py --train-pool <真实银标名池>` 实测逐字背诵率，期望 ≈0。
- **理由**：047 已证真实名增广仅"搬词表=假修"（自有池仍泄漏 10%），根因是背诵训练分布本身；唯有训练分布=真实分布才治本。`--train-pool` 自有池口径（047 泛化的最锋利一刀）防假性归零刷分。
- **被否**：合成+真实混训（重新引入合成名可背，退回泄漏）；仅弃权训练（047 B2 证微效）。

## R4 · 语料规模与 teacher 打标预算

- **决策**：目标非空银标数量级 ~1500–3000（含 ~20% 空样本 → 总 ~2000–4000）。按真实非空率 1/3~1/6 反推，需采集 6k–12k 候选。teacher 打标**逐条本地缓存**（content-hash 键）、**可续跑**，重跑不重复烧配额。
- **理由**：原合成集 10k；域内真实银标不需同量（分布对齐后样本效率更高），但须足量覆盖多引擎形态。缓存+续跑应对 API 限速/中断（Edge Case）。
- **被否**：一次性全量打标无缓存（中断即重烧配额，不可行）。

## R5 · 训练分布 = 服务残差分布

- **决策**：确定性通道（纯 SQL→Calcite、config-driven→规则）能完整覆盖的脚本**不进模型训练集**；模型只训自由命令式/程序化表引用脚本（py/sh/JVM）。
- **理由**：服务态模型只接确定性通道未命中的残差；训练分布对齐服务分布→样本不浪费、指标不被易类稀释。050 已证 config-driven 归规则、SQL 归 Calcite 是最易类。
- **被否**：全类型混训（易类稀释、与服务分布错位）。

## R6 · 方向修正 dir_fix 产品化位置

- **决策**：dir_fix 下沉进 **Python sidecar**（`serve/app.py`）——模型出表集后就地 sqlglot 方向校正，返回方向已修正的边。策略=**dir_fix**（表由模型定、SQL 可识别表方向由 AST 定），**非 override**。
- **理由**：050 实测 dir_fix 方向 0.560→0.596、f1 最优且无 trade-off；override（SQL 整条覆盖）对强模型有害（recall 塌）。放 sidecar 免 Java 侧重写 Calcite 方向逻辑，后端零改动。
- **被否**：Java 侧用 Calcite 做方向（重写成本高、与 050 sqlglot 实现分叉）；union 策略（方向略高但 precision 塌）。
- **健壮性（必移植 050 补丁）**：sqlglot 30.12 对畸形/大片段灾难性回溯爆内存 → 片段窗封顶 800、跳 Jinja/`$CONDITIONS` 模板标记、`update\s+\w+\s+set` 排除 MERGE 内裸 UPDATE、SIGALRM 逐片段限时兜底。

## R7 · 模型规模与升级路径（clarify 定）

- **决策**：3B（Qwen2.5-Coder-3B-Instruct）从干净 base 起训为**达标交付主体**（验收门只卡 3B）；1.5B 同配方作消融对照不设门。配方复用 SEED 20260703/r16 α32/2ep/max2048/4 语言 SYSTEM_PROMPT。单轮未达标→预定义升级：难例挖掘第二轮 → 7B QLoRA 4bit → dir_fix 策略调参，每步重跑同一门。
- **理由**：3B 泄漏最低（10.9%）且 bf16 装 12G（11.9G 已验证）；从 base 起绝不续训合成 checkpoint（否则带入泄漏）。规模单调降泄漏（37→22→11%）。
- **被否**：直接 7B（量化精度不确定、自托管最重，作冲刺备选而非首发）；1.5B 首发（规模不足追平大模型更难）。

## R8 · 2s 服务预算可行性

- **决策**：先实测 3B merged 单脚本延迟；超 2s 则按序调优——MODEL 通道预算单独放宽 / bitsandbytes 8bit 推理提速 / 异步批处理。确定性解码（do_sample=False）保同输入同输出。
- **理由**：SC-008 要求预算内返回；3B 比 1.5B 慢，长脚本+dir_fix 后处理可能逼近 2s。预算是可调优后固定的工程量，非阻塞项。
- **被否**：无脑放宽全局预算（影响其它通道 SLA）。

## R9 · 测试集 B 构建与污染护栏（clarify 定）

- **决策**：新采 teacher 从未参与训练打标的脚本 → agent 逐条证伪初裁 + 维护者抽查终审 → 非空金标 ≥100。训练集与测试集 A（139+141）、B 三者 **content-hash 零重叠**（`build_silver.py` 硬剔除）。合成分数不作判据。
- **理由**：非空 59→≥100 缩窄 CI，回应审稿 small-n 质疑；B 是 teacher 未训练分布的硬证明，防 A 与银标隐性重叠导致的乐观偏差。
- **被否**：仅复用 139+141（n 小 + 隐性重叠）；agent 全权无终审（无人复核可能有偏）。
