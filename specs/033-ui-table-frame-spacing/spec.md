# Feature Specification: 系统设置间距统一 + 全站表格边框包裹

**Feature Branch**: `033-ui-table-frame-spacing`

**Created**: 2026-07-01

**Status**: Draft

**Input**: User description: "项目的Tab主页面的，系统设置所在的Tabs，里面的边距千奇百怪，外边距，内边距统一下，问题2：所有页面的Table，需要看系统设置，里面的项目tab下的table风格，用一个边框包裹起来"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 系统设置各 Tab 间距统一 (Priority: P1)

用户在工作区主页面打开"系统设置"，在其中的用户/角色/项目三个 Tab 间切换。当前每个 Tab 的外边距、内边距、Tab 条与内容之间的间隙、内容区四周留白各不相同（"千奇百怪"），切换时页面元素位置发生跳动、留白宽窄不一，观感杂乱。用户希望三个 Tab 呈现一致、可预期的留白与间距。

**Why this priority**: 这是用户明确点名的第一个问题，直接影响系统设置这一高频入口的专业观感；间距不一致会让整个平台显得未打磨。修复后立即可见、可验证。

**Independent Test**: 仅实现本故事即可交付价值——打开系统设置，逐一切换用户/角色/项目三个 Tab，测量并肉眼确认：内容区四周留白一致、Tab 条到内容的间距一致、切换时无元素位置跳动。

**Acceptance Scenarios**:

1. **Given** 用户打开系统设置视图，**When** 在用户、角色、项目三个 Tab 间切换，**Then** 内容区四周的外边距与内边距在三个 Tab 上完全一致，切换时表格与工具条位置不发生跳动。
2. **Given** 系统设置视图已渲染，**When** 观察 Tab 条与其下方内容区之间的间距，**Then** 该间距为单一、统一的数值，不存在"外层留白 + 内层再留白"造成的双重/叠加空白。
3. **Given** 系统设置视图，**When** 在不同视口宽度（常规桌面尺寸）下查看，**Then** 留白与间距规则保持一致，不出现某一 Tab 明显比其他 Tab 更挤或更松。

---

### User Story 2 - 全站表格采用统一"边框包裹"外观 (Priority: P1)

用户在平台各页面（运维实例、周期/手动工作流、回填、数据源、新鲜度、系统设置等）浏览数据表格。当前不同页面的表格外观不一致：部分被一个带边框的容器"框住"（如系统设置项目 Tab 下的表格），部分则是没有外边框、直接贴在页面上的裸表格。用户希望以"系统设置项目 Tab 下的表格"为参照样式，让**所有页面的表格**都被一个统一的边框容器包裹起来，形成一致的卡片式外观。

**Why this priority**: 用户明确指定这是参照标准并要求"所有页面"统一；表格是平台的主要信息载体，外观一致性对整体质感影响最大。与故事 1 同为用户直接诉求，故并列 P1。

**Independent Test**: 逐一打开含表格的各视图，确认每个表格都被一个统一的边框容器包裹，且该边框样式（边框、圆角、内部留白）与系统设置项目 Tab 下表格的参照样式一致。

**Acceptance Scenarios**:

1. **Given** 系统设置项目 Tab 下的表格被视为参照样式，**When** 打开任意其他含表格的页面，**Then** 该页面的表格同样被一个边框容器包裹，边框、圆角、容器内留白与参照样式在视觉上一致。
2. **Given** 一个当前为"裸表格（无外边框）"的页面，**When** 应用本特性后重新查看，**Then** 表格外层出现统一的边框包裹，且表格的工具条（搜索/筛选/操作按钮）与分页控件仍位于该边框容器之内、布局不被破坏。
3. **Given** 全站任一被包裹的表格，**When** 表格处于加载中、空状态或有数据三种状态，**Then** 边框容器在三种状态下均正确包裹内容，不出现边框缺失、内容溢出或双重边框。

---

### Edge Cases

- **嵌套/内嵌表格**：若某页面在对话框、抽屉或卡片内部已有小型表格，是否需要同样加边框？默认仅统一各视图中的**主数据表格**，内嵌于对话框等既有容器中的从属表格保持原样，避免双重边框。
- **已有 Card 包裹的表格**：系统设置本身已把表格放在带边框的卡片内——统一后不得出现"卡片边框 + 表格边框"叠加的双层边框。
- **空状态与加载态**：表格无数据或加载中时，边框容器仍需存在并保持一致高度，避免容器塌陷或跳动。
- **横向溢出**：列较多导致表格横向滚动时，边框容器应正确裁剪/包裹滚动区域，不被内容撑破。
- **主题切换**：浅色/深色主题下，边框颜色需取自统一的语义化边框色，两种主题下都清晰可辨。

## Requirements *(mandatory)*

### Functional Requirements

#### 系统设置间距统一（故事 1）

- **FR-001**: 系统设置视图的内容区（Tab 条下方承载表格与操作的区域）四周的内边距 MUST 采用单一、统一的取值，三个 Tab（用户/角色/项目）之间完全一致。
- **FR-002**: 系统设置视图 MUST 消除"外层容器留白 + 内层容器再留白"造成的双重/叠加内边距，同一方向的留白只保留一层。
- **FR-003**: Tab 条与其下方内容之间的间距 MUST 为单一统一数值，且在三个 Tab 上一致。
- **FR-004**: 在三个 Tab 之间切换时，表格、工具条、标题等元素的位置 MUST NOT 发生跳动（即三个 Tab 的间距/留白结构相同）。
- **FR-005**: 系统设置各处间距 MUST 取自平台既有的统一间距刻度/设计规范，不得引入一次性的随意数值。

#### 全站表格边框包裹（故事 2）

- **FR-006**: 系统 MUST 以"系统设置项目 Tab 下的表格"为参照，定义一套统一的表格"边框包裹"外观（含外边框、圆角、容器内留白）。
- **FR-007**: 平台所有页面中的主数据表格 MUST 被该统一边框容器包裹，呈现一致的卡片式外观。
- **FR-008**: 表格的工具条（搜索/筛选/操作按钮）与分页控件 MUST 位于边框容器之内，且包裹后其布局与功能不受破坏。
- **FR-009**: 被包裹的表格在加载中、空状态、有数据三种状态下 MUST 均正确显示统一边框，不出现边框缺失、内容溢出或双重边框。
- **FR-010**: 边框颜色 MUST 取自平台统一的语义化边框色，在浅色与深色主题下均清晰可辨。
- **FR-011**: 对已处于带边框容器（如系统设置的卡片）内的表格，统一后 MUST NOT 产生叠加的双层边框。
- **FR-012**: 内嵌于对话框/抽屉等既有容器中的从属小型表格 MAY 不纳入本次统一（默认排除），以避免破坏既有布局与双重边框。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 系统设置的用户、角色、项目三个 Tab，内容区四周内边距的实测值完全相同（三者差异为 0）。
- **SC-002**: 在三个 Tab 间连续切换，表格左上角相对视口的位置偏移为 0（无跳动）。
- **SC-003**: 系统设置视图中，任一方向上不再出现两层叠加的内边距（同方向留白层数为 1）。
- **SC-004**: 全站含表格的页面中，100% 的主数据表格被统一边框容器包裹。
- **SC-005**: 随机抽查任意 3 个不同页面的表格，其边框、圆角、容器内留白与系统设置项目 Tab 参照样式肉眼一致（一致率 100%）。
- **SC-006**: 全站不存在任何一处"双层边框"（卡片边框叠加表格边框）的表格。
- **SC-007**: 浅色与深色两种主题下，被包裹表格的边框均清晰可见（对比度达到可辨识标准）。

## Assumptions

- 参照样式为**系统设置项目 Tab 下表格所在的带边框卡片容器**（边框 + 圆角 + 统一内留白），即用户所述"用一个边框包裹起来"的样板。
- "所有页面的 Table" 指各主视图中承载业务数据的**主数据表格**；对话框/抽屉内的从属小型表格默认不在本次统一范围内。
- 本特性为纯前端外观与布局一致性调整，不改变任何表格的数据、列定义、筛选/分页/操作逻辑与后端接口。
- 间距与边框取值统一从平台既有的设计规范/语义化令牌获取（间距刻度、语义化边框色、圆角规范），不新增一次性硬编码数值；如与设计规范冲突，以设计规范为准并在实现阶段对齐。
- 变更范围限定于工作区内含表格与系统设置的视图组件，不涉及登录页等无表格页面。
