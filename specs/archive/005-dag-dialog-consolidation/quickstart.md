# Quickstart: DAG Dialog Consolidation

**Feature**: 005-dag-dialog-consolidation
**Date**: 2026-06-26

## Prerequisites

- `frontend/` 已安装依赖（`pnpm install`）
- 后端运行中（H2 profile 即可），端口 8000
- 已有至少一个工作流（含已发布 DAG）和至少一个运行过的任务流实例

## Validation Scenarios

### VS-1: 周期 DAG 弹窗（US1 验证）

1. 启动前端: `cd frontend && pnpm dev`
2. 浏览器打开 `http://localhost:4000`，进入 Ops 中心
3. 在"周期任务流"面板中，找到有 DAG 的工作流，点击"查看 DAG"
4. **验证**: 弹窗以 90vw x 90vh 打开，展示 DAG 拓扑，节点无运行时状态圆点
5. 点击 TASK 节点 → 右侧滑出详情面板，展示任务配置（名称、类型、代码、参数）
6. 拖拽分割线 → 面板宽度实时变化，松手后刷新页面重新打开弹窗，**验证宽度被记住**
7. 点击关闭按钮或点击遮罩 → 弹窗关闭

### VS-2: 实例 DAG 弹窗（US1 验证）

1. 在 Ops 中心切换到"任务流实例"列表
2. 找到有 DAG 的实例，点击"查看 DAG"
3. **验证**: 弹窗以相同风格打开，节点显示运行时状态圆点（成功=绿色，失败=红色等）
4. 点击 TASK 节点 → 右侧滑出详情面板，展示实际代码和实际配置（参数已解析）
5. 拖拽分割线 → 面板宽度变化，**验证与周期 DAG 的宽度独立记忆**
6. 关闭弹窗

### VS-3: 实例 DAG 节点高亮（US1 验证）

1. 在"任务实例"列表中，点击某个实例的"查看 DAG"
2. **验证**: 对应任务的节点边框高亮（primary 色）
3. 点击其他节点 → 高亮消失（或保持在原节点，取决于导航意图）

### VS-4: 面板样式一致性（US2 验证）

1. 分别打开周期 DAG 和实例 DAG 的侧面板
2. **验证**: 两个面板的 Header（标题 + 关闭按钮）、DwScroll、section 标题（uppercase tracking-wide）、代码块（Shiki 高亮）风格完全一致
3. 快速切换节点 → **验证**无宽度闪烁（loading 时半透明遮罩覆盖旧内容）

### VS-5: 错误状态

1. 停止后端
2. 打开周期 DAG 弹窗 → **验证**显示错误提示 + 重试按钮
3. 打开实例 DAG 弹窗 → **验证**同样显示错误提示 + 重试按钮
4. 启动后端，点击重试 → **验证**加载成功

### VS-6: Typecheck

```bash
cd frontend && pnpm typecheck
```

**Expected**: 零错误，无未使用的导入警告。

## Regression Check

- [ ] 周期任务流面板"查看 DAG" → 弹窗正常打开、节点展示正确、面板内容正确
- [ ] 任务流实例面板"查看 DAG" → 弹窗正常打开、节点显示运行时状态、面板展示实际代码/配置
- [ ] 面板拖拽宽度在两个弹窗中独立持久化
- [ ] 右键上下文菜单（周期 DAG）仍可正常工作
- [ ] Escape 键（周期 DAG）先关面板再关弹窗
- [ ] 实例 DAG 从任务实例列表跳转时节点高亮
- [ ] 关闭弹窗重新打开后面板状态重置
- [ ] Console 无新增错误/warning
