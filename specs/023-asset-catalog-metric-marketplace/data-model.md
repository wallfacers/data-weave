# Phase 1 Data Model: 资产目录 + 指标市场

权威 DDL = `schema.sql`。本文为设计视图。双方言(PG/H2):`GENERATED BY DEFAULT AS IDENTITY`、JSON 存 `VARCHAR`、枚举 `VARCHAR(32)`、`SMALLINT` 标志、`CONCAT` 不用 `||`、DROP 逆序。

## schema_version 升版

并行特性,合并期定终值;占位 `0.3.0`(假设 021=0.1.0、022=0.2.0 先落)。三处恒等。

## 隔离级别(对齐被引用对象)

- `data_asset` / `metric_listing` / `metric_reuse_ref` = **项目级**(`tenant_id` + `project_id`,对齐 `data_table`/`atomic_metrics`)
- `asset_subscription` = **租户级 + 订阅用户**(订阅是用户行为,可跨项目)
- 可见性 = 隔离 + 敏感度分级权限

## 标签复用

资产标签**复用既有 `tag`/`entity_tag`**(`entity_type='ASSET'`,`entity_id=data_asset.id`),不新建标签表(与任务/工作流标签同机制)。

## 表 1:data_asset(数据资产编目,项目级)

| 列 | 类型 | 说明 |
|---|---|---|
| id | BIGINT IDENTITY PK | |
| tenant_id | BIGINT NOT NULL | |
| project_id | BIGINT NOT NULL | |
| datasource_id | BIGINT NOT NULL | 0=未知,对齐 data_table |
| qualified_name | VARCHAR(512) NOT NULL | 表全限定名,对齐 data_table |
| name | VARCHAR(256) | 展示名 |
| description | VARCHAR(1000) | |
| owner_id | BIGINT | 负责人 |
| steward_id | BIGINT | 数据管家 |
| glossary_terms | VARCHAR(1000) | 业务术语(JSON 数组) |
| sensitivity | VARCHAR(32) DEFAULT 'INTERNAL' | PUBLIC/INTERNAL/CONFIDENTIAL/PII |
| schema_snapshot_json | VARCHAR(4000) | 列/类型快照(对账失效用) |
| lineage_table_ref | VARCHAR(128) | 喂 LineageQueryService 的 tableId(消费血缘) |
| status | VARCHAR(32) DEFAULT 'ACTIVE' | ACTIVE/STALE/RETIRED |
| + 审计列(created_by/updated_by/created_at/updated_at/deleted/version) | | |

**唯一**:`CONSTRAINT uq_data_asset UNIQUE (tenant_id, project_id, datasource_id, qualified_name)`(去重,FR-010)。
索引:`(tenant_id, project_id, status)`、`(tenant_id, project_id, owner_id)`、`(tenant_id, project_id, sensitivity)`。
**关系**:逻辑引用 `data_table(datasource_id, qualified_name)`(不 FK 硬绑,生命周期独立,D1);不复制 `task_table_io` 血缘。

### 状态机(data_asset.status)

```
ACTIVE ──底层表删/改名(schema 对账不一致)──► STALE ──确认下线──► RETIRED
STALE ──底层表恢复──► ACTIVE
```
不变量:STALE 时 UI 不展示"可信绿灯";status 推进可审计。

## 关系图

```
data_asset ──逻辑引用──► data_table(已有)        ──按需消费──► LineageQueryService(neo4j 只读)
data_asset ──质量徽章──► quality_scorecard(份2)   (不可达则降级)
metric_listing ──复用定义──► atomic/derived_metrics(已有)
metric_listing 1──N metric_reuse_ref(防环)
asset_subscription ──变更──► publish AlertSignal(ASSET_CHANGED) ──► 021 消费分发
tag/entity_tag(已有,entity_type='ASSET')──► data_asset 标签
```
