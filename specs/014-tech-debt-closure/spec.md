# Feature Specification: 统一技术债收口（Weft 掉头松散端一次性清理）

**Feature Branch**: `014-tech-debt-closure`

**Created**: 2026-06-29

**Status**: Draft

**Input**: User description: "统一技术债收口（Weft 掉头后的松散端一次性清理）。核心 tasks-as-code 闭环 A–E + 011/012/013 已落地并 E2E 验证健壮，现把之前明确 defer 的松散端正规收口，使 MVP 达到一个干净自洽的里程碑。"

## 背景与动机

DataWeave→Weft 掉头（删服务端 AI、任务=本地文件 + CLI/MCP 往返治理）后，核心 tasks-as-code 闭环（子特性 A–E 文件契约/同步 API/CLI runtime/MCP 重塑 + 011 死代码清理 + 012 占位符迁移 + 013 slug round-trip 保真）已全部落地并经 E2E 闭环真跑验证健壮。

掉头过程中刻意 defer 的若干松散端仍残留，使产品处于"核心稳但边缘有死肉"的状态：服务端-AI 时代的告警 UI 残留、合并遗留的漏迁文案、删源后的构建残骸目录、以及一处需要明确表态的能力缺口。本特性把这批松散端**一次性正规收口**，使 MVP 达到一个干净自洽、可对外演示而无"看着像有实则没有"误导的里程碑。

本特性**不引入任何新运行期能力**——它是纯清理 + 自洽性收口 + 一处缺口的明确文档化裁定。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 移除服务端-AI 时代的"Agent 举手台"残留 UI (Priority: P1)

运维人员打开运维中心（ops 视图），右栏曾有一条"Agent 举手台"告警 rail。该 rail 在 Weft 掉头后已无任何真实数据来源——它原是服务端-AI 时代"agent 主动告警/举手求批"链路的前端展示，当前仅靠开发用 mock 注入（`window.__MOCK_OPS_ALERT__`）喂数，正常运行时永远为空。它的存在让用户误以为系统具备告警/举手能力，实则没有。本故事彻底移除这条残留 rail 及其全部支撑代码与悬空引用，让 ops 视图诚实呈现当前真实能力。

**Why this priority**: 最高优先——这是用户可见的"虚假能力"误导，直接损害产品可信度；且它是孤立死肉，移除无功能风险、收益立竿见影。是本特性 MVP 的核心一半。

**Independent Test**: 仅实现本故事即可独立验证：打开 ops 视图，右栏举手台消失、主舞台布局正常无残缺；前端类型检查零错；全仓搜索无任何指向已删举手台符号的悬空引用。

**Acceptance Scenarios**:

1. **Given** 用户打开运维中心 ops 视图，**When** 视图渲染完成，**Then** 不再出现"Agent 举手台"告警 rail，主舞台/顶条今日大盘照常显示且布局完整无空洞。
2. **Given** 代码库已移除举手台，**When** 运行前端类型检查与构建，**Then** 零错误、零对已删符号（举手台卡片/store/mock 注入器）的悬空引用。
3. **Given** 此前依赖 mock 注入演示告警的开发路径，**When** 举手台移除后，**Then** 该 mock 注入入口（`window.__MOCK_OPS_ALERT__` 及其触发器）一并移除，不留无目标的死注入点。

### User Story 2 - ops 实例面板硬编码中文文案的 i18n 收口 (Priority: P1)

英文 UI locale 下的运维人员在任务流实例 / 周期实例面板里做批量操作时，仍会看到若干**写死的中文**文案（如批量动作按钮、选择上限提示），而面板其余文案均已正确随 UI 语言切换。这是 origin 合并遗留的漏迁。本故事把 ops 区所有残留硬编码中文 UI 文案完整迁入 i18n 文案体系，使其随 UI 语言正确切换，两语言文案集键完全对齐。

**Why this priority**: 与 US1 并列最高——i18n 不完整是用户可见的体验破绽（中英混杂），且违反项目 i18n 三规则①（静态 UI 文案归前端 next-intl，按 UI locale）。是本特性 MVP 的另一半。

**Independent Test**: 仅实现本故事即可独立验证：把 UI locale 切到英文，ops 实例面板的批量动作与提示文案全部呈现英文、无残留中文；两语言文案集键平价校验通过；ops 区全量搜索无遗漏的硬编码中文 UI 串。

**Acceptance Scenarios**:

1. **Given** UI locale = 英文，**When** 用户进入任务流实例面板并多选实例触发批量动作区，**Then** "批量重跑/批量置成功/批量停止"及选择上限提示均显示对应英文，无中文残留。
2. **Given** UI locale = 中文，**When** 同样操作，**Then** 文案显示原中文，语义与迁移前一致。
3. **Given** 选择上限提示含动态数量（当前已选 N 个），**When** 任一语言下展示，**Then** 数量以本地化占位方式正确嵌入文案（非字符串拼接），单复数/语序符合该语言习惯。
4. **Given** 两份语言文案集，**When** 运行键平价校验，**Then** 两集键完全相同，每处文案引用均可静态解析、无缺键。

### User Story 3 - 清理 dataweave-alert 死目录构建残骸 (Priority: P2)

开发者在后端目录里仍看到一个 `dataweave-alert/` 模块目录，但它已无任何源码——只剩一次历史构建产生的 `target/` 残骸（编译 class + jar），且不在根构建模块清单内、其内容不被版本库跟踪。它是服务端-AI 时代 alert 模块删源后未清理的磁盘残骸，徒增认知负担（看着像个模块实则是空壳）。本故事删除该目录，确认后端构建不受影响。

**Why this priority**: 次高——纯磁盘整洁，零功能影响、零版本库影响（目录不被跟踪），但消除"幽灵模块"的认知误导。

**Independent Test**: 删除目录后，后端完整构建通过；根构建模块清单仍只含真实存在的模块；版本库状态不因删除产生任何跟踪文件变更（因其本就未被跟踪）。

**Acceptance Scenarios**:

1. **Given** `dataweave-alert/` 仅含未跟踪的构建残骸，**When** 删除该目录，**Then** 后端完整本地构建（含其余模块编译与安装）成功、无缺失模块报错。
2. **Given** 目录已删，**When** 检视根构建模块清单，**Then** 清单未引用 alert 模块、与删除前一致（本就未引用），无悬空模块声明。

---

### User Story 4 - push 路径表级血缘缺口的明确文档化裁定 (Priority: P3)

平台维护者/未来贡献者需要清楚知道：经 `push` 落库进来的任务**不会**建立表级血缘（当前 JDBC 二部图血缘仅在设计时 `createAndOnline` 路径建立）。这不是 bug 而是**有意延迟**——血缘体系将在发布期改用图数据库（neo4j）重做，届时连同 push 路径血缘一并实现，现在补 JDBC 过渡实现只会被推倒重写。本故事把该缺口作为"已知的、有意的延迟"明确写进架构文档与代码导航注记，杜绝它被误当作遗漏 bug 反复发现/误修。

**Why this priority**: 最低——纯文档化裁定、无代码行为改动，但闭合"功能不闭环"的认知缺口（让缺口可见、可追溯、有定论）。

**Independent Test**: 架构文档血缘小节明确记载"push 不落血缘"为有意延迟及其理由与发布期归宿；代码导航索引的血缘行带有对应注记；读者无需翻代码即可知此缺口的存在与定论。

**Acceptance Scenarios**:

1. **Given** 维护者查阅架构文档血缘小节，**When** 阅读到 push/同步与血缘的关系，**Then** 能看到"push 路径当前不建表级血缘 + 这是有意延迟 + 归宿是发布期 neo4j 图库重做"的明确记载。
2. **Given** 贡献者按代码导航索引定位血缘实现，**When** 读到血缘导航行，**Then** 行内带有指向该已知缺口的注记，不会把"push 不落血缘"误判为待修 bug。

---

### Edge Cases

- **举手台移除波及共享布局**：举手台 rail 若与 ops 视图的三段式布局容器（顶条/主舞台/右栏）耦合，移除右栏后主舞台需自然回填空间，不得留下空栏或错位——验收须含布局完整性目检。
- **举手台符号被非 ops 处引用**：side-panel / backfill-dialog / workflow-canvas 等处若引用了举手台 store 或类型，移除时须连带清理，不得留悬空 import 导致构建失败。
- **i18n sweep 不完整**：除已知 5 处外，ops 区可能还有其它漏迁硬编码中文（含注入到非文本属性如 aria-label/title/placeholder 的）——须全量 sweep，不得只改已知 5 处而漏掉同类。
- **i18n 误伤数据术语**：cron/DAG/SLA/lineage 等数据术语按规约保持英文，不得在迁移中被"翻译"。
- **动态数量插值跨语言**：选择上限提示的"当前 N 个"在英文下语序不同，须用本地化占位机制承载，不得退化为中文式字符串拼接。
- **alert 目录删除的构建依赖误判**：删目录前须确认根构建清单确实未引用它（已确认），避免误删一个仍被引用的模块导致构建断裂。

## Requirements *(mandatory)*

### Functional Requirements

#### US1 — 举手台移除
- **FR-001**: 系统 MUST 从 ops 视图移除"Agent 举手台"告警 rail 及其全部专属前端组件（告警卡片、告警状态存储、右栏容器）。
- **FR-002**: 系统 MUST 移除喂养举手台的开发用 mock 注入机制（`window.__MOCK_OPS_ALERT__` 入口及其触发器/注入器），不留无目标的死注入点。
- **FR-003**: 系统 MUST 清理全仓内对已删举手台符号的所有悬空引用（import/类型/调用），使前端类型检查与构建零错。
- **FR-004**: ops 视图移除右栏后 MUST 保持布局完整——主舞台与顶条照常呈现，无空栏/错位/残缺。

#### US2 — ops i18n 收口
- **FR-005**: 系统 MUST 对 ops 区做硬编码中文 UI 文案的**全量**排查，识别所有未走 i18n 的静态 UI 文案（含按钮文本、提示文本，及 title/aria-label/placeholder 等非正文文本属性）。
- **FR-006**: 系统 MUST 将识别出的硬编码中文 UI 文案迁入前端 i18n 文案体系（按 ops 既有命名空间归类），使其随 UI locale 切换。
- **FR-007**: 两语言文案集 MUST 保持键完全对齐（同键集），每处文案引用可静态解析、无缺键，键平价校验通过。
- **FR-008**: 含动态数量的提示文案 MUST 使用本地化占位机制承载动态值（非字符串拼接），各语言语序/单复数符合习惯。
- **FR-009**: 迁移 MUST 不误伤数据术语（cron/DAG/SLA/lineage/OOM 等保持英文），不改变任一语言下迁移前的语义。

#### US3 — alert 死目录清理
- **FR-010**: 系统 MUST 删除 `dataweave-alert/` 死目录（仅含未跟踪的构建残骸、无源码、不在根构建模块清单内）。
- **FR-011**: 删除后后端完整本地构建 MUST 通过，根构建模块清单 MUST 仍与删除前一致（无悬空模块声明）。

#### US4 — 血缘缺口文档化
- **FR-012**: 系统 MUST 在架构文档血缘小节明确记载"push 路径当前不建表级血缘"为**有意延迟**，含理由（血缘将于发布期改图库 neo4j 重做）与归宿。
- **FR-013**: 代码导航索引的血缘条目 MUST 带有指向该已知缺口的注记，使读者不把它误判为待修 bug。
- **FR-014**: US4 MUST 不包含任何运行期代码行为改动（纯文档/注记）。

#### 全局约束
- **FR-015**: 本特性 MUST NOT 引入新运行期能力、MUST NOT 改动同步 API（pull/push/diff）契约与传输形态、MUST NOT 触碰调度内核与权限/审计内核。

### Key Entities

本特性不引入或修改任何持久化实体、不改数据库 schema。涉及的"对象"均为既有前端模块、文案资源与文档：

- **Agent 举手台 (ops alert rail)**：服务端-AI 时代残留的前端告警展示单元（卡片 + 状态存储 + 右栏 + mock 注入），无真实后端数据源，本特性整体移除。
- **ops i18n 文案资源**：ops 区静态 UI 文案的双语言文案集条目，本特性补齐漏迁项并保持键平价。
- **push 血缘缺口注记**：架构文档与代码导航中关于"push 不落血缘=有意延迟"的文档记载，本特性新增。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ops 视图中"Agent 举手台"rail 出现次数为 0——任意正常运行态打开 ops 视图，均无该 rail。
- **SC-002**: 全仓对已删举手台符号的悬空引用数为 0，前端类型检查与构建零错。
- **SC-003**: 英文 UI locale 下，ops 实例面板（任务流实例 + 周期实例）的批量动作与提示文案中文残留数为 0。
- **SC-004**: 两语言文案集键平价校验 100% 通过（键集完全相同，无单边缺键）。
- **SC-005**: 后端 `dataweave-alert/` 目录不复存在，且删除后后端完整本地构建一次通过。
- **SC-006**: 维护者无需阅读源码，仅凭架构文档即可获知"push 不落血缘"的存在、其为有意延迟、及发布期 neo4j 归宿。
- **SC-007**: 本特性全程零数据库 schema 变更、零同步 API 契约变更、零调度/权限内核改动。

## Assumptions

- **发布前无历史兼容负担**：当前为发布前阶段，举手台移除、文案键调整、目录删除均无需向后兼容历史副本或迁移既有数据。
- **ops 面板已部分 i18n 化**：留存的实例面板其余文案已走 i18n，本次仅补漏迁项，沿用既有 ops 命名空间，无需新建 i18n 基础设施。
- **举手台确为孤立死肉**：已勘察确认其唯一数据源是开发 mock 注入、后端无真告警源；移除不影响任何真实运行链路。
- **alert 目录确为未跟踪残骸**：已勘察确认 `dataweave-alert/` 仅含被忽略的 `target/` 构建产物、0 个版本库跟踪文件、不在根模块清单内；删除为纯磁盘操作。
- **血缘改图库方向已定**：血缘体系发布期改用 neo4j 图库重做的方向已确立，push 路径血缘随之一并实现，故现期不补 JDBC 过渡实现。
- **i18n 三规则与键命名规约沿用**：迁移遵循项目既有 i18n 所有权三规则与键命名/数据术语保留约定，不另立规则。

## 范围边界（Out of Scope）

- 不实现 push 路径表级血缘（US4 仅文档化裁定，实现 defer 至发布期 neo4j）。
- 不为举手台重接真实告警源、不重塑为审批队列（裁定为移除）。
- 不改动 ops 以外区域的 i18n（仅收口 ops 区漏迁）。
- 不新增/重构后端模块（alert 仅删残骸目录，非重建）。
- 不触碰同步 API、调度内核、权限/审计内核、文件契约语义。
