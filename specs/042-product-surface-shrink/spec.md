# Feature Specification: 产品面收缩——移除中台货架页面

**Feature Branch**: `042-product-surface-shrink`

**Created**: 2026-07-03

**Status**: Draft

**Input**: User description: "产品面收缩（direction doc §8/§10 第一步）：按 docs/superpowers/specs/2026-07-03-weft-ai-ops-direction-design.md 的裁决，砍掉中台货架页面 marketplace（指标市场）、reports（报表）、service（数据服务）、integration（数据集成）——移除前端视图、导航入口、视图注册、相关 i18n 键与 redirect；保留 alerts/event-center（待 incident 队列落地后再收敛）、quality/catalog（后续转型另行立项）。"

> 方向依据：[Weft 方向文档 §8 页面清单裁决](../../docs/superpowers/specs/2026-07-03-weft-ai-ops-direction-design.md)。本特性只做"砍"，不做"转型"与"收敛"。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 导航只呈现新方向认可的功能 (Priority: P1)

开发者打开 Weft 工作区，左侧导航中不再出现指标市场、报表、数据服务、数据集成四个入口；其余功能（运维、画布、freshness、血缘、告警、事件中心、目录、质量、数据源、设置等）位置与行为不变。

**Why this priority**: 这是本特性的全部核心价值——让产品面立即与"去中台化"方向一致，用户第一眼看到的产品即是新定位。

**Independent Test**: 以任意角色登录，检查左侧导航的分组与入口清单；逐组核对四个被移除入口不存在、保留入口全部可正常打开。

**Acceptance Scenarios**:

1. **Given** 任意项目角色的用户已登录, **When** 打开工作区, **Then** 左侧导航不出现"指标市场 / 报表 / 数据服务 / 数据集成"任何一项（任一语言下）。
2. **Given** 用户已登录, **When** 逐个点击导航中保留的所有入口, **Then** 每个视图均正常打开，无报错、无空白。
3. **Given** 报表原为默认常驻（pinned）视图, **When** 全新用户首次打开工作区, **Then** 默认常驻标签页中不再包含报表，其余常驻视图（freshness、metrics）不受影响。

---

### User Story 2 - 旧会话与旧链接优雅降级 (Priority: P2)

老用户带着历史痕迹回来：浏览器里存有包含被移除视图标签页的工作区快照，或书签里存有指向被移除视图的深链（如 `/?open=marketplace`）。系统必须优雅降级，不能白屏或报错。

**Why this priority**: 移除功能最常见的回归就是"历史状态炸掉"；不处理会直接阻断老用户进入工作区。

**Independent Test**: 人工构造含被移除视图的本地快照与深链 URL，验证工作区正常加载。

**Acceptance Scenarios**:

1. **Given** 本地保存的工作区快照包含"报表"标签页（且为激活标签）, **When** 用户打开工作区, **Then** 该标签被静默丢弃，工作区以其余标签正常恢复，无错误提示、无控制台异常。
2. **Given** 用户访问深链 `/?open=marketplace`（或 reports/service/integration）, **When** 页面加载, **Then** 回退到默认视图正常展示，不出现白屏。

---

### User Story 3 - 双语文案无残留 (Priority: P3)

被移除功能的界面文案（视图标题、分组标题、占位描述等）从中英两个语言包中同步删除，语言包键集保持一致，且不存在指向已删键的引用。

**Why this priority**: 死键是渐进腐烂的开端；本项目有"两 bundle 键集一致"的 CI 硬约束，必须随删随清。

**Independent Test**: 运行现有 i18n 键集一致性检查与前端类型/测试套件，全部通过。

**Acceptance Scenarios**:

1. **Given** 四个视图已移除, **When** 运行 i18n 键集一致性检查, **Then** 中英语言包键集一致，无孤儿键、无缺失键。
2. **Given** 界面切换到英文, **When** 浏览左侧导航与保留的各视图, **Then** 无缺译占位（如裸键名）出现。

---

### Edge Cases

- 快照中被移除视图恰为激活标签页 → 丢弃后激活标签回退到剩余标签中的合理默认（不允许出现"无激活标签"状态）。
- 导航分组因移除而变空（"分析"组仅含报表）→ 空分组整组不再渲染。
- "资产"分组移除指标市场/集成/服务后仅剩数据源 → 分组保留、仅显示剩余项（分组去留与更名不在本特性范围）。
- 仅拥有 `metric:manage` 权限用于查看指标市场的角色 → 移除后其导航不出现任何空分组或残缺入口；权限种子数据本身不改动。
- 现有导航完整性约束（"入口视图 ∪ 详情视图 = 视图全集"）→ 随视图全集缩减同步收紧，相关自动化校验保持通过。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 从工作区中完全移除以下四个功能视图及其导航入口：指标市场（marketplace）、报表（reports）、数据服务（service）、数据集成（integration）。
- **FR-002**: 系统 MUST 将"报表"从默认常驻（pinned）视图集合中移除；其余常驻视图行为不变。
- **FR-003**: 恢复历史工作区快照时，系统 MUST 静默丢弃指向已移除视图的标签页；若被丢弃者为激活标签，MUST 将激活态回退到剩余标签，不得报错或出现空激活态。
- **FR-004**: 访问指向已移除视图的深链（`/?open=<view>`）时，系统 MUST 回退到默认视图，不得白屏或抛出用户可见错误。
- **FR-005**: 系统 MUST 从中英两个语言包中同步删除仅被这四个视图使用的文案键，保持两包键集一致，且所有保留界面的文案引用可静态解析。
- **FR-006**: 系统 MUST 同步更新导航分组：因移除而变空的分组不再显示；部分移除的分组仅显示剩余项。
- **FR-007**: 系统 MUST NOT 改动任何服务端能力：相关后端接口、MCP 工具、权限种子数据、数据库结构一律保持原样（服务端收缩另行立项）。
- **FR-008**: 系统 MUST NOT 影响保留视图（含 alerts、event-center、quality、catalog）的现有行为——它们的收敛与转型明确不在本特性范围。

### Key Entities

- **工作区视图（View）**: 工作区内一个可打开的功能页面，具有导航入口、标题文案、可选权限门槛与常驻属性；本特性将视图全集从 18 个缩减到 14 个。
- **工作区快照（Snapshot）**: 浏览器侧持久化的标签页集合与激活态；可能包含指向已移除视图的陈旧条目。
- **导航分组（Nav Group)**: 左侧导航的有序功能分组；成员随视图全集缩减而调整。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 左侧导航入口总数从 16 减少到 12，且任一语言下检索不到四个被移除功能的名称。
- **SC-002**: 携带含被移除视图的旧快照或旧深链进入工作区的用户，100% 正常进入（无白屏、无用户可见错误）。
- **SC-003**: 全部现有自动化门槛（前端类型检查、单元测试、i18n 键集一致性检查）保持通过。
- **SC-004**: 保留的全部视图逐一打开验证可用，行为与移除前一致。

## Assumptions

- 后端（接口、MCP 工具、`metric:manage` 等权限码、种子数据、schema）全部保留不动；"服务端指标/报表域退役"是后续独立特性。
- alerts / event-center 的"收敛为 incident 队列"、quality / catalog 的"转型"均为方向文档 §10 的后续步骤，本特性不触碰。
- "资产"分组仅剩数据源后是否更名或并组，留给后续导航整理，本特性最小化处理（保留分组、仅显剩余项）。
- 被移除视图无服务端会话状态需要清理；历史快照清理仅发生在浏览器侧恢复逻辑中。
