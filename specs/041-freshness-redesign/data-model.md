# Data Model: 数据新鲜度页面重新设计

**Feature**: 041-freshness-redesign
**Date**: 2026-07-03

## New Tables

### freshness_daily_snapshot — 项目级每日分布快照

每天凌晨拍摄一次，记录全项目的新鲜度分布聚合。

| Column | Type | Constraint | Description |
|--------|------|------------|-------------|
| id | BIGINT | PK, GENERATED BY DEFAULT AS IDENTITY | 主键 |
| tenant_id | BIGINT | NOT NULL | 租户 id |
| project_id | BIGINT | NOT NULL | 项目 id |
| snapshot_date | DATE | NOT NULL | 快照日期 |
| total_tasks | INT | NOT NULL DEFAULT 0 | 当日总任务数 |
| fresh_count | INT | NOT NULL DEFAULT 0 | FRESH 档位任务数 |
| aging_count | INT | NOT NULL DEFAULT 0 | AGING 档位任务数 |
| stale_count | INT | NOT NULL DEFAULT 0 | STALE 档位任务数 |
| never_count | INT | NOT NULL DEFAULT 0 | NEVER 档位任务数 |
| created_at | TIMESTAMP | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**Unique**: `(tenant_id, project_id, snapshot_date)`

**Purpose**: 概览区日环比计算——当天分布 vs 前一天分布 → delta 和百分比变化。

### freshness_task_daily — 每任务每日档位快照

每天凌晨拍摄一次，记录每个任务当天的新鲜度档位。

| Column | Type | Constraint | Description |
|--------|------|------------|-------------|
| id | BIGINT | PK, GENERATED BY DEFAULT AS IDENTITY | 主键 |
| tenant_id | BIGINT | NOT NULL | 租户 id |
| project_id | BIGINT | NOT NULL | 项目 id |
| task_id | BIGINT | NOT NULL | 任务 id |
| snapshot_date | DATE | NOT NULL | 快照日期 |
| tier | VARCHAR(16) | NOT NULL | 档位：FRESH / AGING / STALE / NEVER |
| age_hours | BIGINT | NULL | 距今小时数（NEVER 为 NULL） |
| created_at | TIMESTAMP | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**Unique**: `(tenant_id, project_id, task_id, snapshot_date)`
**Index**: `idx_ftd_task_dates ON (task_id, snapshot_date DESC)` — 支持按任务查最近 N 天趋势

**Purpose**: 表格 7 天火花图数据源——查某任务最近 7 天 tier 值，映射为火花图 Y 轴。

## Extended Entities (No Schema Change)

### TaskFreshness（扩展领域模型）

现有 `FreshnessService.FreshnessRow` 扩展以下字段：

| Field | Type | Source | Description |
|-------|------|--------|-------------|
| workflowName | String | `workflow_def.name` via JOIN | 所属工作流名 |
| scheduleType | String | `workflow_def.schedule_type` | MANUAL / CRON / DEPENDENCY |
| scheduleHuman | String | 计算字段（Java 方法） | 人读调度周期，如"每天 06:00" |
| trend7Days | int[] | `freshness_task_daily` JOIN | 最近 7 天档位数值数组，[4,3,2,1,...] |

### 火花图 Y 轴映射

```
FRESH → 4 (y=4)
AGING → 3 (y=8)
STALE → 2 (y=12)
NEVER → 1 (y=16)
```

数值越大档位越好，Y 轴越低位置越好（SVG 坐标系原点左上）。

## SQL JOIN Chain

```
task_def td
  LEFT JOIN workflow_node wn ON wn.task_id = td.id
  LEFT JOIN workflow_def wd ON wd.id = wn.workflow_id AND wd.deleted = 0
  LEFT JOIN LATERAL (
    SELECT tier, snapshot_date
    FROM freshness_task_daily ftd
    WHERE ftd.task_id = td.id
    ORDER BY ftd.snapshot_date DESC
    LIMIT 7
  ) trend ON true
  LEFT JOIN task_instance ti ON ti.task_id = td.id AND ti.deleted = 0
```
