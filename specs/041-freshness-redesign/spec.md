# Feature Specification: 数据新鲜度页面重新设计

**Feature Branch**: `041-freshness-redesign`

**Created**: 2026-07-03

**Status**: Draft

**Input**: User description: "数据新鲜度页面头脑风暴，表格字段性格的展示重新设计，现在看起来就和一个demo没啥区别"

## Clarifications

### Session 2026-07-03

- Q: 调度周期人读映射覆盖范围？ → A: 覆盖 6 种核心模式（每分钟、每 N 分钟、每小时、每天 HH:mm、每周 N HH:mm、每月 D HH:mm），其余 fallback 显示原始 Cron 表达式。
- Q: 火花图 < 7 个数据点时的渲染行为？ → A: 自适应宽度——有几个数据点就画几个点占满 SVG，不固定 7 槽位。视觉上需精致、不平淡。
- Q: 进度条警告阈值？ → A: 固定 70%（非 80%），避免大部分项目常态在 70-85% 时持续触发警告。
- Q: 订阅告警的通知渠道？ → A: 复用现有 AssetSubscription + 告警系统基础设施，新鲜度作为新增订阅类型接入已有通知渠道。
- Q: 概览区卡片布局？ → A: 横排 4 列等宽卡片（新鲜 | 偏旧 | 陈旧 | 从未成功），与运营中心顶条统计卡片布局一致。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 新鲜度全局概览 (Priority: P1)

运维打开数据新鲜度页面，页面顶部展示全局概览区，包含各时效档位的任务计数、日环比变化趋势、整体数据健康度进度条以及分布比例。运维无需翻页或筛选即可快速判断"当前数据是否健康"。

**Why this priority**: 概览区是页面第一屏的核心信息层，直接回答运维最关心的问题——有多少任务产出是陈旧/偏旧的、整体状况比昨天变好还是变差。这是新鲜度页面从"demo 感"升级为专业运维工具的关键。

**Independent Test**: 打开数据新鲜度页面，概览区四个统计卡片（新鲜/偏旧/陈旧/从未成功）显示项目级计数；进度条直观展示健康占比；日环比箭头和百分比反映变化方向；分布文字列出各档位比例。

**Acceptance Scenarios**:

1. **Given** 项目中有 178 个任务，其中 128 个新鲜、35 个偏旧、12 个陈旧、3 个从未成功，**When** 运维打开数据新鲜度页面，**Then** 概览区显示：4 个计数卡片（128/35/12/3）、数据健康度进度条填充 72%、分布文字"新鲜 72% · 偏旧 20% · 陈旧 6.7% · 从未成功 1.7%"。
2. **Given** 昨天有快照数据可作为对比基准，**When** 页面加载概览区，**Then** 每个计数卡片下方显示日环比箭头（↑/↓/→）和百分比变化。
3. **Given** 该项目昨天没有快照数据（首次使用），**When** 页面加载概览区，**Then** 统计卡片不显示日环比信息，仅显示当前计数值。
4. **Given** 项目新鲜任务占比低于 70%，**When** 页面渲染进度条，**Then** 进度条颜色变为警告色，提示运维关注。

### User Story 2 - 增强表格查看任务级新鲜度 (Priority: P2)

运维在概览区下方看到任务列表，每行包含：任务名及所属工作流、调度周期、时效档位 Badge、最近成功时间、距今时长、7 天趋势火花图。表格默认按最陈旧优先排列，运维可一眼定位问题任务。

**Why this priority**: 表格是概览区的下钻层——看到"陈旧 12 个"后自然想看到具体是哪些任务。增强列让每行信息密度足以支撑运维判断（这个任务是"该跑没跑"还是"跑了但慢了"）而无需点进详情。

**Independent Test**: 表格默认按 NEVER → STALE → AGING → FRESH 排序，同档位最陈旧在前。每行显示调度周期、火花图等新增信息。可通过筛选器和预设快速缩小范围。

**Acceptance Scenarios**:

1. **Given** 表格加载完成，**When** 运维查看任务行，**Then** 每行包含 7 列：任务名（含工作流名副标题）、调度周期、时效 Badge、最近成功时间、距今时长、7 天趋势火花图、操作图标。
2. **Given** 任务有 N 天历史数据（1 ≤ N ≤ 7），**When** 渲染该行的火花图，**Then** 显示 N 个数据点的自适应宽度折线图，当天点实心、历史点空心，反映该任务的新鲜度档位变化趋势。
3. **Given** 任务是新建的（无历史数据），**When** 渲染该行，**Then** 火花图列显示"—"。
4. **Given** 默认为"最陈旧优先"排序，**When** 运维切换为"最新鲜优先"，**Then** 排序立即反转，FRESH 任务排在最前。
5. **Given** 运维在筛选器中选择"陈旧"分档，**When** 筛选生效，**Then** 表格仅显示时效档位为 STALE 的任务，概览区突出对应计数卡片。

---

### User Story 3 - 时效 Badge 点击联动筛选 (Priority: P3)

运维在概览区点击某个时效档位的计数卡片，或在表格中点击某行的时效 Badge，筛选器自动切换为该档位，表格过滤到仅显示该档位的任务。运维无需手动操作筛选下拉框。

**Why this priority**: 消除"看到→想筛选→去工具栏找下拉"的摩擦。点击联动让概览区卡片成为可交互的快捷入口，时效 Badge 成为行内筛选触发器。

**Independent Test**: 点击概览区"陈旧 12"卡片 → 表格筛选为仅陈旧任务，工具栏筛选下拉选中"陈旧"。点击某行"偏旧"Badge → 同样联动。

**Acceptance Scenarios**:

1. **Given** 概览区显示"陈旧: 12"，**When** 运维点击该卡片，**Then** 表格筛选器自动选中"陈旧"分档，表格仅显示 12 行陈旧任务。
2. **Given** 表格某行时效 Badge 为"偏旧"，**When** 运维点击该 Badge，**Then** 筛选器将"偏旧"追加到已选分档，表格过滤为仅偏旧任务。
3. **Given** 时效 Badge 已被选为当前筛选条件，**When** 运维再次点击同一 Badge，**Then** 取消该分档筛选，恢复全量视图。

---

### User Story 4 - 行操作快捷入口 (Priority: P4)

每行末尾提供图标操作按钮（查看日志、查看实例、查看 DAG、重跑、订阅告警），鼠标悬浮显示 tooltip 说明。运维可直接在新鲜度页面对问题任务执行操作，无需跳转到其他页面再搜索。

**Why this priority**: 新鲜度页面本质是"问题发现→问题处理"的起点。如果发现问题任务后还需要跳转+搜索才能操作，页面就只是清单而非工作台。操作按钮把发现和处理闭环在一屏内。

**Independent Test**: 点击行末"重跑"图标 → 弹出确认弹窗 → 确认后提交重跑。点击"查看日志"图标 → 打开该任务的实例日志面板。

**Acceptance Scenarios**:

1. **Given** 表格中有一个"从未成功"的任务，**When** 运维 hover "重跑"图标，**Then** 显示 tooltip"重跑"，点击后弹出确认弹窗，确认后提交重跑请求。
2. **Given** 运维 hover "查看日志"图标，**When** 点击，**Then** 打开该任务最近一次实例的日志面板。
3. **Given** 运维 hover "订阅告警"图标，**When** 点击，**Then** 注册该任务的新鲜度告警订阅，图标变为已订阅态。

---

### User Story 5 - 每日快照与历史对比 (Priority: P5)

系统在每日凌晨自动拍摄所有任务的新鲜度快照，保存每任务每日档位和项目级聚合分布。概览区的日环比、表格中的 7 天趋势火花图均基于此快照数据。

**Why this priority**: 快照是 P1/P2 的数据基础。没有历史快照就没有趋势和环比。P5 优先级最低仅因为它是后台自动化任务——用户无感知、不影响页面交互，但页面其他能力的完整性依赖它。

**Independent Test**: 快照表在每日凌晨 02:00 后存在当天数据；dashboard API 返回 trend 字段可计算日环比；task API 返回 trend7Days 数组可渲染火花图。

**Acceptance Scenarios**:

1. **Given** 系统已运行超过 1 天，**When** 每日凌晨 02:00 后查询快照表，**Then** 每个 (tenant, project, task) 有一条当日记录，包含档位和距今小时数。
2. **Given** 快照数据已积累 7 天，**When** 请求任务列表 API，**Then** 每个任务的 trend7Days 数组长度为 0-7，按日期升序排列。
3. **Given** 某个日期快照因系统异常未生成，**When** 页面请求 dashboard，**Then** trend 字段为 null，概览区不显示日环比，不报错。

---

### Edge Cases

- 项目无任何任务时 → 概览区所有计数显示 0，进度条 0%，分布文字"暂无数据"，表格显示空状态（图标 + 提示文字）。
- 所有任务都是 FRESH → 进度条 100% 填充，颜色正常（primary），日环比可能为 0 或正增长。
- 所有任务都是 NEVER → 进度条 0%，颜色为警告色；火花图全部留空。
- 快照尚未启用（无历史数据）→ 日环比和火花图显示"—"，概览区不报错。
- 同时多项目切换 → 概览区和表格数据随项目隔离刷新（复用现有项目级响应式机制）。
- 超过 90 天的旧快照 → 定时任务自动清理，不影响查询性能。
- 定时快照任务执行中系统发生切换 → 写入操作幂等（不产生重复数据）。

## Requirements *(mandatory)*

### Functional Requirements

**概览区**

- **FR-001**: 页面顶部 MUST 横排展示四个等宽时效档位统计卡片（新鲜 | 偏旧 | 陈旧 | 从未成功），布局与现有运营中心顶条统计卡片一致。每个卡片显示档位名称、当前计数、对应语义色图标。
- **FR-002**: 统计卡片 MUST 在有前一天快照时显示日环比变化，变化方向用箭头（↑/↓/→）和百分比表示。
- **FR-003**: 统计卡片 MUST 可点击，点击后将对应档位设为表格筛选条件。
- **FR-004**: 概览区 MUST 包含一条数据健康度进度条，反映新鲜任务占总任务的比例，视觉样式与现有 Worker 节点资源使用进度条一致。
- **FR-005**: 进度条 MUST 在新鲜占比低于 70% 时切换为警告色。
- **FR-006**: 概览区 MUST 在进度条下方显示各档位占比的分布文字。

**增强表格**

- **FR-007**: 表格 MUST 新增"调度周期"列，以人读格式展示。覆盖 6 种核心 Cron 模式（每分钟、每 N 分钟、每小时、每天 HH:mm、每周 N HH:mm、每月 D HH:mm）；MANUAL 类型显示"手动"；未能匹配的 Cron 表达式显示原始值作为 fallback。
- **FR-008**: 表格 MUST 新增"7 天趋势"列，以迷你折线图展示该任务最近 7 天的新鲜度档位变化趋势。火花图宽度自适应实际数据点数量（1-7 个点），不固定 7 槽位。折线需平滑，当天数据点用实心标记，历史点用空心标记，整体观感精致。无历史数据时显示"—"。
- **FR-009**: 表格中时效 Badge MUST 可点击，点击后联动筛选器过滤对应档位。
- **FR-010**: 任务名列 MUST 在任务名下方显示所属工作流名（副标题样式）。
- **FR-011**: 表格 MUST 保留现有筛选器（时效分档多选、任务名搜索）和预设。

**行操作**

- **FR-012**: 每行末尾 MUST 提供图标操作按钮组：查看日志、查看实例、查看 DAG、重跑、订阅告警，交互样式与现有运营中心实例面板操作列一致。
- **FR-013**: 每个操作图标 MUST 在 hover 时显示 tooltip。
- **FR-014**: "重跑"操作 MUST 在提交前弹出确认弹窗。
- **FR-015**: "订阅告警"操作 MUST 支持切换（点击订阅/再次取消），当前订阅态有视觉区分。订阅后端对接现有 AssetSubscription 和告警通知基础设施，不新建独立通知渠道。

**后端快照**

- **FR-016**: 系统 MUST 在每日凌晨 02:00 自动对所有任务拍摄新鲜度快照，记录每任务当日档位和距今小时数。
- **FR-017**: 系统 MUST 同时聚合项目级分布快照（各档位计数和总任务数）。
- **FR-018**: 快照数据 MUST 保留最近 90 天，过期数据由同一定时任务清理。
- **FR-019**: 快照写入 MUST 幂等（同日多次执行不产生重复记录）。
- **FR-020**: 任务列表查询 MUST 返回每任务的最近 7 天趋势数组，供渲染火花图。
- **FR-021**: 概览区数据查询 MUST 返回当前分布和日环比信息（无快照时趋势字段置空）。

### Key Entities

- **每日任务快照（Freshness Task Daily）**: 代表单个任务在某一天的新鲜度状态记录，包含任务标识、快照日期、档位（FRESH/AGING/STALE/NEVER）、距今小时数。按 (任务, 日期) 唯一。
- **每日分布快照（Freshness Daily Snapshot）**: 代表某个项目在某一天的新鲜度分布聚合，包含项目标识、快照日期、总任务数、各档位计数。按 (项目, 日期) 唯一。
- **任务新鲜度（Task Freshness）**: 代表某任务当前时刻的新鲜度状态（已有概念），本次扩展增加：所属工作流、调度类型、人读调度周期、近 7 天档位趋势。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 运维可在 2 秒内从概览区判断当前项目数据新鲜度的整体健康状况（通过进度条和分布比例）。
- **SC-002**: 运维可在单屏内完成"发现陈旧任务 → 点击 Badge 筛选 → 点击操作按钮处理"的完整闭环，无需跳转搜索。
- **SC-003**: 通过火花图和日环比，运维可快速区分"该跑没跑"（调度周期 vs 距今时长）和"跑了但慢了"（趋势持续下滑），两类场景的定位时间从分钟级缩短到秒级。
- **SC-004**: 快照任务不增加页面加载延迟——概览区和表格数据的加载时间与当前页面持平。

## Assumptions

- 现有任务定义已包含调度类型和周期表达式的字段，可直接用于生成人读调度周期。
- 现有任务定义与工作流定义的关联可直接获取工作流名。
- 现有 Worker 节点资源使用进度条的视觉组件可直接复用为数据健康度进度条。
- 现有运营中心实例面板的操作按钮模式（icon + tooltip + confirm dialog）可直接复用。
- 日环比以每日分布快照的前一天记录为基准；前一天快照缺失时环比不可用（不报错）。
- 火花图展示最近 7 天；任务创建不足 7 天则展示实际天数。
- 90 天保留期覆盖 7 天趋势 + 日环比 + 潜在的未来月度趋势扩展需求。
- 快照定时任务复用现有调度基础设施，不引入独立的调度框架。
