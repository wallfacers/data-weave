# Feature Specification: 血缘小模型实证研究——可投论文加固

**Feature Branch**: `065-lineage-paper`

**Created**: 2026-07-12

**Status**: Draft

**Input**: User description: "血缘小模型接下来几个目标奔着发论文去。头条=泄漏曲线+诚实去偏+分层信封（不靠超 teacher，绕开 n≈49）；脊椎=泄漏科学，脚本血缘降为动机、信封降为落地。补齐评审必看的方法学缺口：统计诚实层、工具基线、可复现 benchmark。不新增 GPU/凭据/人工标注。列级血缘 defer 为 future work。"

## 背景与意图 *(context)*

041/047/052/054/059/063 已完成三个核心发现并合入 main：① 逐规模记忆泄漏曲线（0.5/1.5/3B 逐字泄漏 37→22→11%、合成 0.99 vs 真实 0.24–0.32 塌缩）；② 诚实去偏（来源隔离 + content-hash 去污染 + CV 暴露样本内校准乐观）；③ 分层复核信封。**发现本身已存在，本特性只做把它们变成一篇能过审论文所缺的方法学加固**，不改模型、不重训。

评审反馈指出的可发表资产是这三条实证线，唯一真硬伤是「头条若为 3B 超 teacher，则 n≈49 非空 gold 撑不起显著性、且 gold 系 teacher 派生（循环论证）」。本特性通过**重定头条**（避开"超 teacher"）+ **补三个方法学缺口**消解该硬伤，使论文可投 MSR（备 EMSE）。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 统计诚实层：每个数字带置信区间 (Priority: P1)

作为论文作者，我需要现有评测产出的每个头条数字都带 95% 置信区间，并对「3B vs teacher 的 precision 差」做配对显著性检验，**诚实报告其在当前样本量下是否显著**——把"n≈49 撑不起超越"这把评审的枪，用诚实的统计陈述先发制人地拆掉，而不是回避。

**Why this priority**: 这是整个头条重定位的落脚点，也是唯一能拆掉核心硬伤的一块。没有它，任何比较类陈述都会被审稿人以"无显著性"驳回；有了它，"teacher 对比在此 n 下不显著"反而成为诚实性加分。是 MVP：单独交付即可让论文在统计严谨性上站住。

**Independent Test**: 对现有落盘的逐脚本 counts（gold C / gold A 上的 tp/fp/fn），运行新增的重采样与配对检验模块，产出每个指标的点估计 + 95% CI，以及 3B-vs-teacher 配对检验的 p 值与判定；无需 GPU、无需新数据即可离线复算并断言输出结构正确。

**Acceptance Scenarios**:

1. **Given** 一批逐脚本 counts（precision/recall/F1 的 tp/fp/fn 列表），**When** 运行统计诚实层，**Then** 每个聚合指标返回点估计 + 95% 置信区间，且区间随样本量收敛（可用重复种子验证确定性）。
2. **Given** 同一评测集上 3B 与某 teacher 的逐脚本对错分布，**When** 运行配对显著性检验，**Then** 输出配对统计量与 p 值，并给出"在此样本量下（不）显著"的明确判定。
3. **Given** 当前 n≈49 的非空 gold C，**When** 检验 3B vs teacher precision 差，**Then** 结果被如实标注为"样本量受限、结论不显著"，并可直接引入论文的诚实性陈述。

---

### User Story 2 - 工具基线：LLM vs 现有工具，脚本上的结构性失效 (Priority: P2)

作为论文作者，我需要在同一评测集上加入**确定性/规则类工具基线**（而非只有 LLM-vs-LLM），产出"现有工具 vs 学习模型"的对照——尤其证明**在命令式脚本（Python/Shell/Spark）上确定性 SQL 解析工具接近零召回，而学习模型能救回**，为"为何需要学习式抽取器"提供实证动机。

**Why this priority**: 审稿人想看的是"LLM vs 现有工具"，不是"LLM vs teacher LLM"。脚本上工具≈0/模型救回的对照图，是脊椎（泄漏科学）之下、支撑"脚本血缘领域动机"的关键证据。次于 P1，因为它增强说服力但不拆核心硬伤。

**Independent Test**: 把工具基线接入现有评测入口，在 gold C 与 gold A 上与学习模型同尺子跑分；断言在纯 SQL 子集上工具可比、在脚本子集上工具召回≈0 而模型显著高于工具，产出对照表与分层数字。

**Acceptance Scenarios**:

1. **Given** gold C（含脚本样本），**When** 运行规则/确定性工具基线与学习模型，**Then** 得到同尺子的 precision/recall/F1 对照表，脚本子集单列。
2. **Given** 纯 SQL 子集，**When** 对比工具与模型，**Then** 工具在结构化 SQL 上召回可比（验证工具基线本身有效、非稻草人）。
3. **Given** 命令式脚本子集，**When** 对比工具与模型，**Then** 工具召回≈0（结构性失效），模型召回显著高于工具，差值带 P1 的置信区间。

---

### User Story 3 - 可复现 benchmark：第三方能复算头条 (Priority: P3)

作为论文读者/复现者，我需要一份公开、自包含的评测基准与脚本，使我**无需私有凭据**即可复算论文头条表与招牌图（泄漏曲线、合成 vs 真实塌缩、工具 vs 模型脚本对照），从而信任结论。

**Why this priority**: 复现性是评审的第二条必项，公开可复现 > 大而私有。它把"语料/权重 gitignored 走 HF"的复现折扣补上。列 P3 因为它依赖 P1/P2 产出的评测物先稳定。

**Independent Test**: 从发布物（评测集 + 评测脚本 + 基线 + 复现说明）在一台无项目私有凭据的机器上，按文档步骤复算出与论文一致的头条表与至少一张招牌图。

**Acceptance Scenarios**:

1. **Given** 发布的评测基准与脚本，**When** 第三方按复现说明执行，**Then** 复算出的头条指标与论文一致（在 P1 报告的 CI 内）。
2. **Given** 无项目私有凭据的环境，**When** 运行复现流程，**Then** 全流程不依赖受限凭据即可完成（受凭据限制的部分被显式隔离、可跳过）。
3. **Given** 发布物，**When** 检查其内容边界，**Then** 明确标注哪些数据公开、哪些因泄漏/许可考量不公开，且不泄漏训练集使评测失效。

---

### User Story 4 - auto-gold 适度扩容（可选/稳健性） (Priority: P4)

作为论文作者，我可选地把非空评测 gold 从 ~49 适度扩容，以收紧泄漏曲线/信封数字的置信区间，**但明确标注该扩容为稳健性证据、仍系 teacher 派生、非独立真值**。

**Why this priority**: 在本头条下非 blocker（曲线显著性不依赖它）。仅当 P1–P3 落地后仍有余量、且投稿周期允许时执行。默认可不做。

**Independent Test**: 复用现有自动 gold 流水线扩容后，重跑 P1 的 CI，确认关键区间收窄；产物在论文中被标注为"稳健性、teacher 派生"。

**Acceptance Scenarios**:

1. **Given** 现有自动 gold 流水线与可用凭据，**When** 扩容非空 gold，**Then** 得到更大规模的评测集且关键指标 CI 相对收窄。
2. **Given** 扩容后的 gold，**When** 在论文中呈现，**Then** 附带"teacher 派生、非独立真值、仅作稳健性"的显式声明。

---

### Edge Cases

- 当某评测子集样本极少（如脚本子集 n 很小）时，置信区间会很宽——必须如实呈现宽区间，不得因难看而隐藏或缩小口径。
- 当配对检验的两模型在多数样本上并列（tie）时，配对检验须正确处理并列，不得把并列当作差异。
- 当工具基线在某类脚本上偶发抽出（非严格零）时，须报告真实非零值，不得为了"工具≈0"叙事而清零。
- 当复现环境缺少受限凭据（如某 teacher 的 API key）时，复现流程须能跳过该分支并仍复算出不依赖它的核心结论。
- 当扩容 gold 引入分布漂移（新样本与旧集不同源）时，须区分"因样本更多而收窄"与"因分布变化而移动"，避免把漂移当作稳健性。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 对现有评测的每个聚合指标（precision/recall/F1/方向准确率）在脚本级重采样基础上产出 95% 置信区间，且在给定固定种子时结果可确定性复现。
- **FR-002**: 系统 MUST 支持对两个模型在同一评测集上的逐脚本对错分布做配对显著性检验，输出配对统计量、p 值与"是否显著"判定，并正确处理并列样本。
- **FR-003**: 系统 MUST 能在不修改既有打分逻辑（表名集合口径的 tp/fp/fn 计算）的前提下叠加上述统计层——统计层只消费既有逐脚本 counts，不改变任何指标定义。
- **FR-004**: 系统 MUST 在同一评测集上纳入至少一个非学习式（规则/确定性解析）工具基线，与学习模型同尺子产出对照，并支持按"纯 SQL 子集 / 命令式脚本子集"分层报告。
- **FR-005**: 系统 MUST 量化并报告"命令式脚本子集上工具基线召回≈0、学习模型显著更高"的差距，且该差距附带 FR-001/FR-002 的区间与显著性。
- **FR-006**: 系统 MUST 产出一份自包含、无需项目私有凭据即可运行的可复现评测发布物（评测集 + 评测脚本 + 基线 + 复现说明），第三方据此复算的头条指标须落在论文报告的置信区间内。
- **FR-007**: 系统 MUST 在发布物中明确标注数据公开边界（哪些公开、哪些因泄漏/许可不公开），且发布不得包含使真实集评测失效的训练数据泄漏。
- **FR-008**: 系统 MUST 将受限凭据（如某 teacher API）所门控的评测分支与核心可复现路径解耦——核心结论的复算不得依赖受限凭据。
- **FR-009**: 系统 MAY（可选）复用既有自动 gold 流水线适度扩容非空评测集；若执行，产物 MUST 被显式标注为"teacher 派生、非独立真值、仅作稳健性"。
- **FR-010**: 论文中每一条比较类/数值类头条陈述 MUST 关联到本特性产出的一个可复现证据项（CI、显著性判定、对照表或招牌图），不得出现无证据支撑的比较陈述。
- **FR-011**: 系统 MUST 在论文中显式声明 out-of-scope 项（列级血缘 = future work；大规模独立人工 gold 未做，以来源隔离/去污染/CV 缓解；受凭据限制的 teacher 三方对比），作为诚实性陈述而非隐藏。

### Key Entities *(include if feature involves data)*

- **评测集（gold）**：带表级 reads/writes 标签的脚本样本集合（gold A 人工裁决 / gold C GitHub-fresh 自动一致）；关键属性=样本量、非空数、来源、是否含脚本子集；列级字段存在但 out-of-scope。
- **逐脚本 counts**：每个样本的 tp/fp/fn/方向 计数，是统计层与打分层之间的稳定接口。
- **模型/基线预测**：学习模型（0.5/1.5/3B）与工具基线（规则/确定性）在评测集上的落盘预测，供离线复算与配对检验。
- **统计证据项**：置信区间、配对检验结果、分层对照表、招牌图——论文每条头条陈述的支撑单元。
- **发布物（benchmark）**：公开的评测集 + 脚本 + 基线 + 复现说明，含数据公开边界声明。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 论文中 100% 的头条数值/比较陈述都关联一个带 95% 置信区间或显著性判定的可复现证据项（无裸数字比较）。
- **SC-002**: "3B vs teacher precision 差"在当前样本量下的（不）显著性被显式报告，且该诚实陈述被纳入论文，而非回避。
- **SC-003**: 至少一个确定性工具基线在纯 SQL 子集上召回与既有报告可比（证明基线有效），在命令式脚本子集上召回≈0，且学习模型在脚本子集上召回显著高于该工具（差值附区间）。
- **SC-004**: 一名不具备项目私有凭据的第三方，按发布的复现说明，能在一次尝试内复算出与论文一致的头条表（落在 SC-001 的区间内）。
- **SC-005**: 泄漏曲线（逐字泄漏 vs 规模）与合成 vs 真实塌缩，作为可从发布 harness 复现的招牌图呈现，数值与论文一致。
- **SC-006**（仅当执行 FR-009）：非空评测集扩容后，至少一项关键指标的 95% 置信区间宽度相对收窄，且扩容在论文中被标注为 teacher 派生的稳健性证据。

## Assumptions

- 目标投稿为 MSR（主）/ EMSE（备）；具体投稿周期截止时间待外部确认，影响排期与 FR-009 取舍，但不影响本特性可构建范围。
- 三个核心发现（泄漏曲线、去偏、信封）已完成并合入 main，本特性不重训模型、不改模型架构。
- 统计层以脚本级 bootstrap 置信区间 + McNemar 配对检验实现（既有 `metrics.py` 的逐脚本 counts 与 `aggregate` 求和为稳定接口，统计层叠加于其上）。
- 工具基线以既有 `regex_baseline`（补跑至 gold C）+ 新增 SQLLineage 为默认；Calcite 等库级/跨进程工具基线默认不做（代价高），列为可选。
- 发布物默认公开"仲裁后 gold C + 评测脚本 + 基线 + 复现说明"；合成 held-out 是否随发布公开取决于泄漏考量，默认**不公开合成训练相关集**以免使真实集评测失效。
- 可用凭据：qwen-max（m1）与 m2 回退可用于打标；GitHub/HF token 可用于自动 gold 扩容；deepseek（m3）凭据缺失，其 teacher 三方对比不在关键路径（脚注说明）。
- 列级血缘（column-level）显式 out-of-scope，作为论文 future work；其现状已在 `docs/superpowers/specs/2026-07-11-lineage-leak-paper-hardening-design.md` 记录。
- 全部工作落 `ml/lineage-extractor/`，隔离于 065 worktree（`../dw-065-lineage-paper`），不触碰后端/前端/其他特性。
