# Feature Specification: DAG 节点详情侧面板

**Feature Branch**: `004-dag-node-detail-panel`

**Created**: 2026-06-26

**Status**: Draft

**Input**: "002 speckit 实现了运营中心，任务流查看DAG，但是往往在发布的DAG阶段，运维人员喜欢的是点击，或右击操作DAG的节点查看节点配置，节点配置一般是代码和配置相关（后续的一些任务不一定有代码） 你需要将，弹框的窗体分成1/4 右侧，用于查看发布任务的信息"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 点击节点查看任务配置信息 (Priority: P1)

运维人员在 Ops 中心点击任务流的"查看 DAG"按钮，弹框展示已发布版本的 DAG 图。运维人员点击 DAG 中任意任务节点，弹框右侧滑出约 1/4 宽度的详情面板，展示该节点关联任务的发布时配置信息——包括任务名称、类型、发布版本号、执行代码/脚本内容（如任务类型包含代码）、配置参数列表。运维人员无需离开 DAG 弹框即可快速了解节点的线上配置。

**Why this priority**: 这是本功能的核心价值——运维人员在排查问题或巡检时，最频繁的操作就是查看某个节点"线上跑的是什么代码/配置"。在 DAG 弹框中直接点节点看配置，比跳转到任务编辑页再切回来高效得多，是运维日常最高频需求。

**Independent Test**: 在周期任务流列表中点击 ONLINE 状态任务流的"查看 DAG"→ 弹框展示 DAG → 点击某个任务节点 → 右侧面板展示该节点任务的配置信息。可独立于右击菜单和面板缩放功能完成端到端测试。

**Acceptance Scenarios**:

1. **Given** DAG 弹框已打开并展示已发布版本的 DAG 图，**When** 运维人员点击 DAG 中的某个 TASK 类型节点，**Then** 弹框右侧滑出宽度约 1/4 的详情面板，面板内展示该节点关联任务的配置信息。
2. **Given** 右侧面板已展示节点 A 的信息，**When** 运维人员点击 DAG 中的另一个 TASK 节点 B，**Then** 右侧面板内容切换为节点 B 的信息（就地更新，无关闭-重开动画）。
3. **Given** DAG 弹框打开且右侧面板已可见，**When** 运维人员点击 DAG 画布的空白区域（非节点区域），**Then** 右侧面板关闭，DAG 图恢复占满弹框。
4. **Given** 该节点关联的任务包含执行代码/脚本，**When** 查看右侧面板，**Then** 面板中展示该任务的代码/脚本内容（只读模式，按语言语法高亮）。
5. **Given** 该节点关联的任务不包含代码（如纯配置类任务），**When** 查看右侧面板，**Then** 面板中代码区域标注"该任务类型无执行代码"，配置参数正常展示。

### User Story 2 - 右击节点弹出快捷操作菜单 (Priority: P2)

运维人员在 DAG 弹框中右击某个任务节点，在鼠标位置弹出上下文菜单，提供"查看任务详情"选项——点击后与 P1 行为一致，打开/激活右侧面板并展示该节点信息。

**Why this priority**: 右击菜单是桌面运维工具的通用交互范式，有经验的运维人员会直觉式地右击节点。它为后续扩展更多节点快捷操作（如跳转实例运行记录、查看日志等）预留交互入口。优先级低于 P1 因为左键点击即可覆盖核心需求。

**Independent Test**: DAG 弹框中右击任务节点 → 弹出上下文菜单 → 点击"查看任务详情" → 右侧面板展示该节点信息。可独立于左键点击功能完成验证。

**Acceptance Scenarios**:

1. **Given** DAG 弹框已展示 DAG 图，**When** 运维人员右击某个 TASK 类型节点，**Then** 在鼠标位置弹出上下文菜单，至少包含"查看任务详情"选项。
2. **Given** 上下文菜单已弹出，**When** 运维人员点击"查看任务详情"，**Then** 菜单关闭，右侧面板展示该节点关联任务的配置信息。
3. **Given** 上下文菜单已弹出，**When** 运维人员点击菜单外的任意位置或按 Escape 键，**Then** 上下文菜单关闭。
4. **Given** 运维人员右击 VIRTUAL 类型节点（占位节点），**When** 上下文菜单弹出，**Then** 不包含"查看任务详情"等任务相关选项（虚拟节点无关联任务）。

---

### User Story 3 - 面板交互与尺寸适配 (Priority: P3)

右侧面板打开后，运维人员可通过拖拽面板左侧分割线调整面板宽度（范围约 1/5 到 1/3 弹框宽度，最小 280px）。面板内部信息分区展示（基本信息区和代码/配置区），长内容区内部可滚动。面板右上角有关闭按钮，关闭后 DAG 恢复占满整个弹框。

**Why this priority**: 面板交互的舒适度影响使用频率和长时间工作的疲劳度。面板在长代码内容和小屏幕下需要合理的空间管理。它依赖 P1 的面板基础框架，但不阻塞核心功能上线。

**Independent Test**: 打开右侧面板 → 拖拽分割线调整宽度 → 滚动长代码内容 → 点击关闭按钮关闭面板 → 再次点击节点重新打开面板。可独立于内容获取逻辑完成交互测试。

**Acceptance Scenarios**:

1. **Given** 右侧面板已打开，**When** 运维人员拖拽面板左侧分割线，**Then** 面板宽度在约 1/5 到 1/3 弹框宽度之间连续变化，最小宽度不低于 280px。
2. **Given** 右侧面板已打开，**When** 运维人员点击面板右上角关闭按钮，**Then** 面板关闭，DAG 图恢复占满整个弹框。
3. **Given** 弹框宽度不足 900px，**When** 运维人员点击节点打开面板，**Then** 面板以 320px 固定宽度打开，DAG 区域相应缩小（面板不隐藏或覆盖 DAG）。
4. **Given** 面板中代码内容较长（超过 500 行），**When** 运维人员查看面板，**Then** 代码区内部可纵向滚动，面板本身随弹框等高。

---

### Edge Cases

- 节点关联的任务已被删除（taskId 对应的任务不存在）时，右侧面板显示"关联任务已删除"提示，不崩溃或白屏。
- 节点关联任务的发布版本与当前最新版本不同时，面板展示**发布时冻结**的任务版本信息，并标注发布版本号（而非当前草稿版本）。
- 连续快速点击不同节点（3 次/秒以上）时，面板稳定展示最后一次点击的节点信息，不出现闪烁或中间状态残留。
- 点击 VIRTUAL 类型节点（占位节点）时，面板不触发（虚拟节点无关联任务，不展示面板也不展示菜单）。
- 后端返回的节点详情数据缺失某些字段（如代码为空、配置参数为空），面板对应区域显示占位提示而非空白。
- 网络请求失败（获取节点详情超时或返回错误）时，面板内显示错误提示和重试按钮，DAG 图本身不受影响，可继续浏览。
- Escape 键行为：面板打开且上下文菜单未弹出时，按 Escape 关闭面板（不关闭弹框）；面板已关闭时，按 Escape 关闭整个 DAG 弹框。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在 DAG 弹框中支持点击 TASK 类型节点，点击后在弹框右侧展示该节点关联任务的配置信息面板。
- **FR-002**: 右侧面板宽度默认为弹框宽度的 1/4，最小宽度 280px。
- **FR-003**: 右侧面板必须展示任务的配置信息：任务名称、任务类型标识、发布时版本号、执行代码/脚本内容（如适用）、配置参数列表（键值对）。
- **FR-004**: 对于不包含代码的任务类型，面板中代码区域必须标注"该任务类型无执行代码"，不得展示空白代码块。
- **FR-005**: 点击 DAG 画布空白区域（非节点区域）时，右侧面板必须关闭，DAG 图恢复占满弹框。
- **FR-006**: 连续点击不同 TASK 节点时，面板内容必须平滑切换为新节点信息，无中间状态闪烁。
- **FR-007**: 右击 TASK 节点时必须弹出上下文菜单，至少包含"查看任务详情"选项；点击该选项后打开右侧面板展示该节点信息。
- **FR-008**: 右击 VIRTUAL 节点时，上下文菜单不得包含任务相关选项。
- **FR-009**: 右侧面板必须提供关闭按钮，点击后面板关闭。
- **FR-010**: 面板左侧分割线必须支持拖拽调整宽度，范围在弹框宽度的 1/5 到 1/3 之间，最小不低于 280px。
- **FR-011**: 节点关联任务已被删除时，面板必须显示友好提示，不得白屏或抛出异常。
- **FR-012**: 节点详情数据请求失败时，面板应显示错误信息并提供重试按钮，DAG 图本身不受影响。
- **FR-013**: 面板中的代码内容必须以只读方式展示，配有对应语言的语法高亮。
- **FR-014**: 面板打开时按 Escape 键先关闭面板；面板已关闭时按 Escape 键关闭弹框。
- **FR-015**: 节点详情数据必须来源于该任务流**发布时冻结的任务版本快照**，确保展示内容与线上运行时一致。

### Key Entities

- **节点任务详情 (Node Task Detail)**：在 DAG 弹框中选中 TASK 节点时展示的任务快照数据。包含：任务名称、任务类型标识、发布版本号、执行脚本文本（可选，任务类型决定是否有）、配置参数集合（键值对）。数据来源于任务流发布时冻结的任务版本快照。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 运维人员从打开 DAG 弹框到看到某个节点的配置信息，操作步骤不超过 2 步（点击"查看 DAG" → 点击节点）。
- **SC-002**: 点击节点后右侧面板展示内容的时间不超过 1 秒（含数据获取和渲染，适用于代码 500 行以内的节点）。
- **SC-003**: 面板宽度拖拽调整流畅，无可见延迟或卡顿。
- **SC-004**: 连续切换 10 个不同节点，面板无闪烁、错位或内容残留。
- **SC-005**: 运维人员在首次使用时无需额外引导即可发现并成功查看节点配置（可用性测试通过率 ≥ 90%）。

## Assumptions

- DAG 弹框（来自 002-ops-dag-viewer）已实现并可用，作为本功能的基础容器和入口。
- 后端 `published-dag` API 返回的节点数据（taskId、taskVersionNo、name、nodeType 等）已可用；获取任务版本详细配置（代码内容、配置参数）的 API 若有不足，在实现阶段补充。
- "任务类型是否包含代码"由系统任务类型元数据决定——部分任务类型为纯配置驱动，无代码。
- 代码语法高亮的语言由任务类型映射（如 SQL→SQL、Python→Python、Shell→Bash）。
- 右击菜单交互面向桌面端运维人员，移动端不考虑。
- 节点关联任务的删除为软删除或快照保留场景；若出现快照中 taskId 物理删除导致数据不可用，面板展示降级提示。
