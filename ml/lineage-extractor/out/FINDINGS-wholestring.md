# 整串 exec-sink 抽取升级——修 CTE 泄漏 + 抬自动采纳层（测试集 A / held-out）

**日期**：2026-07-08 · **改动**：`channel_router` exec-gated 路径从**关键字锚定片段**改为**整串抽取**
（从执行 sink `spark.sql("…")`/`.execute(…)`/CLI `-e`/赋值 抓完整字符串实参喂 sqlglot）

## 动机

052 收尾时定位到 Tier 1 exec-gated precision ~0.68 的头号残余=**CTE 片段泄漏**：关键字锚定
在 `INSERT` 处起片段、丢掉 `WITH cte AS(...)` 前缀 → `FROM cte` 被当读表。语句级 CTE 排除
被片段化击穿（当时记为已知残差）。**整串抽取保住 `WITH` 前缀后，`find_all(exp.CTE)` 复得别名并扣除。**

## 结果：全轴改善，自动采纳层达治理完美

| 指标 | 整串前（片段锚定） | 整串后（整串抽取） |
| --- | --- | --- |
| Tier 1 exec-gated precision（全集） | 0.488 | **0.511** |
| Tier 1 exec-gated precision（非空） | 0.677 | **0.706** |
| Tier 1 exec-gated recall（非空） | 0.127 | **0.145** |
| Tier 1 幻觉率 | 0.000 | 0.000 |
| Tier 1 自动覆盖非空真表 | 23.4% | **27.7%** |
| **一致层 precision** | 0.952 | **1.000** |
| **一致层 recall** | 0.142 | **0.163** |
| 并集召回 | 0.837 | 0.837（模型主导，SQL 改善落在已覆盖内） |
| 复核负载 | 2.64 候选/脚本 | 2.66 候选/脚本 |

- **CTE 泄漏结构性修复**：`WITH cte AS(...) INSERT … FROM cte` 不再吐 `cte`；多行三引号 SQL 整串正确解析（新增单测锁定）。
- **一致层 precision 0.952→1.000**：更干净的 Tier 1 使双通道一致层零误报 → 自动采纳完全治理安全。
- **一致层 recall +2.1pt**：自动采纳覆盖更多真表 = 复核者人力再省一点。
- 并集召回天花板 0.837 不变（由模型主导，SQL 通道改善落在其已覆盖区内）——升级点在**自动采纳的精度与覆盖**，非并集召回。
