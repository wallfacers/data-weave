# 分级置信度校准——把二元门升级为分级可靠性曲线（测试集 A / held-out）

**日期**：2026-07-08 · **改动**：新增 `realeval/confidence_calibration.py`——给每条候选血缘边打
**部署期可得**（不看金标）的置信信号，按经验 precision 校准分级，产出「自动采纳到第 k 级」操作点。

## 动机

052 部署信封是**二元门**：一致层（SQL∩模型）自动采纳（precision 1.000 但召回仅 0.170），
其余并集候选一股脑进复核队列（2.66 候选/脚本，**无序**）。两个残缺：
① 复核队列无排序，复核者无法「从最可能对的先看 / 早停」；
② 单一一致门可能把本可安全自动采纳的高置信边也压进人工。

## 部署期置信信号（逐候选边，均不依赖金标）

- **通道归属**：SQL∩模型一致 / SQL-only / 模型-only（一致 = 双通道 canon 互认）；
- **名字限定性**：`schema.table`（限定）vs 裸名（更易是别名/临时/解析碎片）；
- 候选边在 canon 下合并互斥（`t` 与 `db.t` 不重复计数）。

## 逐级可靠性（gold A，非空 59 脚本 / 141 真表，canon 口径）

| 置信级 | 候选边数 | 经验 precision |
| --- | --- | --- |
| 一致（SQL∩模型） | 23 | **1.000** |
| SQL-only·限定名 | 0 | — |
| SQL-only·裸名 | 11 | **0.273** |
| 模型-only·限定名 | 37 | **0.919** |
| 模型-only·裸名 | 84 | 0.690 |

## 关键发现：先验「SQL 恒优于模型」被证伪，限定性才是主信号

校准出的经验 precision 排序 = **一致 → 模型限定名 → 模型裸名 → SQL 裸名**。

- **SQL-only·裸名 precision 仅 0.273**——最低。这些裸名多是 CTE 别名/临时表/解析碎片，
  非持久血缘。041-R「有 SQL 时解析器严格优于模型」的结论**只对限定名成立**。
- **模型-only·限定名 precision 0.919**——近治理级。当模型吐出 `schema.table` 形态的名字，
  它极可能是真表。
- **限定性（schema.table）是跨通道的主置信信号**，强于「哪个通道产出」。

## 部署收益:校准序抬高治理阈下的自动采纳召回

先验序把 SQL 裸名（0.273）排在模型级之前 → 过早污染累计前沿，卡死在一致层（零提升）。
按经验 precision **重排**后：

| 级序 | 自动采纳至 | 采纳边数 | precision | 召回 | 复核(候选/脚本) |
| --- | --- | --- | --- | --- | --- |
| 基线(仅一致层) | 一致 | 23 | 1.000 | 0.170 | 2.24 |
| **校准前沿** | **+模型限定名** | **60** | **0.950** | **0.411** | **1.61** |
| （续采纳） | +模型裸名 | 144 | 0.799 | 0.823 | 0.19 |

- **治理阈 precision ≥ 0.95 下**：自动采纳召回 **0.170 → 0.411（2.4×）**，
  复核负载 **2.24 → 1.61 候选/脚本**，precision 仍守 0.950。
- 复核队列按**校准序**排序（一致 → 模型限定 → 模型裸名 → SQL 裸名）→ 复核者从最可能对的
  先看、按治理阈早停。
- 对照：先验序前沿召回仅 0.170——**校准的价值即在于用经验证据纠正级序**。

## 边界与零破坏

- **样本内校准诚实披露**：calibrated 级序在同一 gold A 上既定序又评估，141 表小样本。
  定性结论（限定性主导、先验序证伪）稳健；精确前沿点（0.411 / 0.95）应在 held-out
  测试集 B 复核后再固化——这正是下一杠杆「测试集 B」的用武之地。
- 纯函数、无 torch（模型预测从 `dump_model_preds` 的 jsonl 读入）→ 可无 GPU 单测（+7 单测）。
- 未改 `channel_router` 抽取逻辑、未碰已发布 HF 模型 / 后端 sidecar / 053·055 目录接地面
